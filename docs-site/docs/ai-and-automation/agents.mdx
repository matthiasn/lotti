# Run and monitor agents

Agents let Lotti carry a defined responsibility across multiple wake cycles.
Each running instance comes from an agent template, uses an inference setup,
and keeps its own state, reports, observations, and activity history. Open
**Settings → Agents** to see the system as a whole.

Agent reports, observations, and queued wakes describe the agent's working
state; they do not silently rewrite the personal logbook. When a task agent
wants to change task data, Lotti presents a proposal for your review. See
[Review proposed changes](../organize-and-reflect/tasks.mdx#review-proposed-changes)
for the per-task approval controls and their narrow initial-title and language
exceptions.

Agent token usage is reported together with all other AI work in [Review AI usage and impact](./usage.mdx).

## Inspect the running instances

Open **Instances** to see every agent and evolution session. Rows expose the
agent kind, lifecycle, assigned template, and soul. Grouping by soul is useful
when several responsibilities share one voice and decision style.

Select an instance to inspect its internals. The **Stats** tab shows model
usage, its template, current inference route, lifecycle controls, wake count,
and scheduling state.

The remaining instance tabs preserve different evidence:

- **Reports** contains the agent's saved report snapshots.
- **Conversations** groups messages by wake cycle.
- **Observations** exposes captured findings without mixing them into reports.
- **Activity** shows the chronological message and tool trail.

**Pause** keeps the instance but stops normal work. **Re-analyze** asks a task
agent to reconsider its current subject. **Destroy** is destructive and should
only be used when the instance itself is no longer wanted; changing the
template for future agents does not require destroying unrelated instances.

## Understand wake cycles

Open **Wake Cycles** to see work that is running now and work due later. A
**Pending** wake was requested by the agent. A **Scheduled** wake was placed by
the system or planner. Each row keeps the linked task, project, or day visible
alongside the responsible agent and a live countdown.

Deleting a queued wake removes that timer; it does not delete the agent or its
linked task. If a wake immediately returns, inspect the agent state or the
automation that requested it instead of repeatedly deleting the symptom.

## Roadmap: check your understanding, not just your output

The agent roadmap includes learning quizzes generated from your own tasks.
This feature is planned; it is not available in the app yet.

From any task you will be able to tap **Check my understanding** and get a
short quiz tailored to that task's content — study notes get comprehension
and application questions, engineering work gets mechanism and debugging
questions. Quizzes mix multiple-choice questions with open questions you can
answer by typing or by voice, with an editable transcript before you submit.

Grading is conversational: when an answer is shallow or ambiguous, the AI asks
a follow-up before judging, the way a tutor would. Every missed item comes
with a concrete explanation of what you missed, grounded in the task content
the question came from, and the session ends with a score, a label, and a gap
list. Grades are clearly presented as AI feedback about one quiz — never a
credential, ranking, or comparison with other people — and quiz history stays
private, syncable, and deletable like other agent data.

The planned progression is:

1. **Quiz any task** on demand, with a quick-check or deep-dive depth choice
   and fresh questions each time.
2. **More surfaces and gentle suggestions** — projects, days, and agent
   reports, plus an optional non-modal "Quiz yourself on this?" card after
   meaningful completions, with strict caps and snooze/disable controls.
3. **Consented workspace evidence** so engineering quizzes can draw on
   repository diffs, symbols, and test summaries through a read-only
   integration.
4. **History insights** that surface recurring gap themes over time.

The design is evidence-bounded: questions and grades come only from content
Lotti actually captured, and thin content yields fewer questions rather than
filler. Suggestions never run AI in the background before you accept, and
quizzes never block completing a task.
