## The building blocks, without pretending they are all the same

| Building block | Use it for | Example |
| --- | --- | --- |
| Category | A meaningful area of life or work | Health, Family, Research |
| Task | An outcome that needs progress | Prepare the V1 release |
| Checklist | Concrete steps inside a task | Verify release notes |
| Entry | Context or evidence | Note, image, audio recording |
| Planned block | A proposed slot in the day | Release review, 10:00–11:00 |
| Time record | What actually happened | Release review, 10:18–11:07 |
| Habit | A recurring behavior you evaluate | Take medication |
| Measurable | A numeric observation | Resting heart rate |
| Dashboard | A view over useful measurements | Training overview |

## Agents are guests, not co-owners

An agent can read the context you give it, form an opinion, summarise a messy
task, and propose a useful next change. It does not get to silently edit your
record. Proposed task, checklist, status, or date changes wait for you to
confirm or dismiss them. The narrow exceptions are an initial title or language
for an otherwise empty task—after that, the same human-approval rule applies.

This is not politeness bolted onto a prompt. Lotti keeps the personal logbook
and agent working memory separate, so the agent cannot treat your history as a
scratchpad. You can use AI, tune it, or leave it entirely alone.

## A practical first orbit

1. Create categories that you genuinely use to make decisions.
2. Capture work as tasks and keep supporting material attached to them.
3. Use Daily OS to decide what belongs in a particular day.
4. Record actual time and observations without forcing them to match the plan.
5. Review habits and dashboards for patterns rather than individual perfect
   days.

## Keep ownership clear

- A linked task owns its title and category.
- A Daily OS block owns its placement and duration.
- A time entry owns the record of what happened.
- AI proposes; you review and decide.

:::tip The point is not an immaculate timeline

If the plan changed, let it change. If the day went sideways, record what did
happen. A useful system helps you see the shape of your life; it does not ask
you to perform a more productive one for the database.

:::
