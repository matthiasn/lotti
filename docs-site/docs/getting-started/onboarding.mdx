# Set up Lotti with onboarding

Onboarding connects the minimum pieces needed for Lotti's AI-assisted task
workflow: one provider, a recording style, at least one category, and a first
real task. It is a guided setup, not a separate demo environment—providers,
categories, and tasks created here remain in the normal app.

Open **Settings → Onboarding** whenever you want to start or replay the flow.
The page also shows whether this profile has already created its first
AI-structured task.

## Start with the welcome promise

<ManualScreenshot
  caseId="onboarding/welcome"
  alt="Lotti onboarding welcome panel inviting the user to connect an AI brain and turn speech into a structured task"
  caption="The welcome panel opens over the current app. Choose your AI brain to continue, or leave the flow without changing setup."
/>

Select **Choose your AI brain** to begin. Choosing **Look around first**,
tapping the dimmed area outside the panel, and using the system back action
all leave onboarding without connecting anything. You can return from Settings
later.

Lotti may also offer this welcome automatically to an eligible profile. The
automatic prompt has a limited retry cadence; the Settings entry does not, so
you always have a way back.

## Choose where task structuring runs

<ManualScreenshot
  caseId="onboarding/providers"
  alt="Lotti onboarding provider chooser with Melious.ai, Mistral, Gemini, Qwen, OpenAI, and Ollama"
  caption="Primary hosted providers appear first. Expand More options for OpenAI and the local Ollama path."
/>

The provider page identifies the available hosted and local choices:

- **Melious.ai** is the recommended EU-hosted default with dynamic model
  routing.
- **Mistral**, **Gemini**, and **Qwen** expose their corresponding hosted
  regions.
- **OpenAI** is available under the expanded options.
- **Ollama** connects to a compatible local endpoint and does not require a
  hosted API key.

<ManualScreenshot
  caseId="onboarding/api-key"
  alt="Lotti onboarding provider verification panel showing a verified local Ollama connection"
  caption="Lotti verifies the chosen provider before enabling Connect. Hosted providers use the same step with an API-key field."
/>

For a hosted provider, paste the provider API key. Lotti verifies the
connection before enabling **Connect**. Ollama verifies the configured local
endpoint and reports reachable models. A successful connection saves the
provider and seeds the models and inference profile used by the remaining
steps.

<ManualScreenshot
  caseId="onboarding/success"
  alt="Lotti onboarding confirmation that the AI provider is connected and ready"
  caption="The connection confirmation is intentionally calm; the larger payoff is reserved for the first real task."
/>

Captured task text is sent for structuring to the provider assigned to the
selected category. Choose a provider whose deployment and data-handling model
fits the material you plan to capture. A reachable local Ollama endpoint keeps
that structuring request on the machine running it.

## Pick the recording feel and work areas

<ManualScreenshot
  caseId="onboarding/recording-style"
  alt="Lotti onboarding recording-style chooser with Modern energy orb and Analogue VU meter options"
  caption="Choose the recording visual that feels clearest. Both options use the same capture pipeline."
/>

After connection succeeds, choose the recording style used by the first-task
capture and later voice-capture surfaces. You can change it again under
**Settings → Recording Style**.

<ManualScreenshot
  caseId="onboarding/categories"
  alt="Lotti onboarding work-area chooser with Work, Fitness, Family, Friends, and Add your own choices"
  caption="Choose one or more areas for the new provider. Scaled text and narrow layouts automatically use one full-width choice per row."
/>

Next, select one or more work areas. Presets cover common categories such as
Work, Fitness, Family, and Friends; **Add your own** creates a category with a
name that matches your workspace. Onboarding reuses an existing category with
the same name instead of creating a duplicate, restores it when necessary, and
connects it to the newly seeded inference profile.

The manual fixture creates two real destinations—**Penguin Operations** and
**Mission Control**—so the next step demonstrates the same category choice a
working account sees.

## Capture the first real task

<ManualScreenshot
  caseId="onboarding/first-task"
  alt="Lotti first-task onboarding panel with voice capture, Penguin Operations and Mission Control destinations, suggestions, and Rather type"
  caption="Choose where the task should land, then speak naturally, use a starter suggestion, or select Rather type. Penguin Operations is selected here."
/>

The first-task panel uses Lotti's normal capture pipeline. Choose the target
category, then use one of three paths:

1. Select the recording visual and speak the task in a normal sentence.
2. Choose a starter suggestion to try the structuring path immediately.
3. Choose **Rather type?** and enter the thought without using the microphone.

Lotti echoes the captured words while it structures them. The result is a real
task in the chosen category, not onboarding-only sample data. When structuring
produces checklist suggestions, they arrive on the task as proposals for you
to confirm instead of changing the task silently.

<ManualScreenshot
  caseId="onboarding/task-created"
  alt="Lotti onboarding success panel showing the created task Inspect the Europa sardine relay"
  caption="The typed Project Waddle thought becomes “Inspect the Europa sardine relay.” The task stays in the panel until you choose it."
/>

Choose the created task card to close onboarding and open its normal task page.
From there you can review checklist proposals, add cover art or evidence, set a
due date and priority, and bring the task into Daily OS.

If you close the flow after connecting a provider, Lotti keeps that provider
setup and treats the welcome as completed. Replaying onboarding later remains
safe: it is an explicit Settings action rather than another automatic prompt.
