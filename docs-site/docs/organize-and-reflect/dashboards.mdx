# Build useful dashboards

A dashboard groups charts that answer a recurring question. **Colony
operations** brings habitat pressure, sardine demand, and expedition headcount
together; **Mission readiness** keeps a private launch review separate. Build
around a decision rather than collecting every available number.

## Create a dashboard

1. Open **Settings → Dashboard management**.
2. Select **Create dashboard**.
3. Add a clear name and a description of the question it should answer.
4. Optionally assign a category and privacy setting.
5. Add and arrange charts, then save.

<ManualScreenshot
  caseId="dashboards/settings-detail"
  alt="Colony operations dashboard editor assigned to Penguin Operations with privacy and active options"
/>

- **Category** makes the dashboard available to category-based filtering.
- **Private** hides the dashboard unless private entries are shown.
- **Active** controls whether it appears in the Dashboards list without
  deleting the definition.

## Compose and order charts

<ManualScreenshot
  caseId="dashboards/charts"
  alt="Colony operations chart list with Habitat pressure daily average, Sardines consumed daily sum, and Penguins accounted for daily sum"
  caption="Drag with the handle to reorder. Select the adjustment control on a measurement chart to change its aggregation, or remove it with ×."
/>

Chart order is dashboard order. Put the signal that frames the rest first. For
Colony operations, habitat pressure leads because an unsafe habitat changes how
every other number should be interpreted.

Measurement charts keep their own aggregation on the dashboard item. That lets
the same measurable appear with a different summary in another dashboard
without changing the underlying observations.

## Add charts from supported sources

<ManualScreenshot
  caseId="dashboards/sources"
  alt="Dashboard editor offering Habits, Measurements, Health, Surveys, and Workouts as chart sources plus JSON export"
/>

The editor can add:

- **Habits** for completion history.
- **Measurements** for user-defined numeric observations.
- **Health** data supported by the current platform and permissions.
- **Surveys** for configured questionnaire results.
- **Workouts** imported from supported health sources.

Expand a source, select one or more charts, and add them. Availability depends
on the definitions and platform data that exist; health permission prompts and
supported types vary by device.

## Open and interpret a dashboard

Open **Insights** to see active dashboards. The list shows each dashboard's
category icon, name, and purpose. Use the filter in the header to focus on one
or more categories.

<ManualScreenshot
  caseId="dashboards/list"
  alt="Lotti Insights dashboard list with Colony operations and private Mission readiness dashboards"
  caption="Desktop reserves the right pane for the selected dashboard; mobile opens the selected dashboard as a separate page."
/>

Select **Colony operations** to open its charts. On desktop, the list and
selected dashboard remain side by side. On mobile, the dashboard opens as its
own page with a back button.

<ManualScreenshot
  caseId="dashboards/view"
  alt="Colony operations dashboard showing 90 days of habitat pressure and sardines consumed with the dashboards list beside it on desktop"
  caption="The fixed header keeps the 30, 90, 180, and 365-day range controls visible while the charts scroll below."
/>

The selected range applies to every chart on the page. The chart cards retain
their own units and aggregation: habitat pressure uses a daily average while
sardine consumption and penguin headcount use daily sums. Select the **+** on a
measurement card to record a new observation of that type.

<ManualScreenshot
  caseId="dashboards/view-crew"
  alt="Colony operations dashboard scrolled to Sardines consumed and Penguins accounted for charts"
  caption="A brief dip to 36 penguins is visible in the 90-day crew history; the surrounding days return to the expected 37."
/>

Treat a chart as evidence, not a verdict. Check the aggregation, units, date
range, missing observations, and privacy state before drawing conclusions.

## Export dashboard configuration

**Save and copy JSON** saves the dashboard, then copies its configuration
and referenced measurable definitions. This is useful for inspection or moving
a configuration, but it is not a live sync link: later edits do not update an
already copied payload.
