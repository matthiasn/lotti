# Work with tasks

Tasks group an outcome with the context and steps needed to finish it. Keep
titles outcome-oriented and put supporting notes, images, recordings, and time
records inside the task.

<ManualScreenshot
  caseId="tasks/workspace"
  alt="Production task workspace showing real task rows with cover-art thumbnails from the Interplanetary Penguin Logistics demo data"
/>

On desktop, the task list remains visible beside the selected task so you can
move through work without losing context. On mobile, selecting a task opens its
detail view full-screen. Cover art follows the task everywhere it appears,
including the task list and linked Daily OS agenda items.

<ManualScreenshot

/>

The same task can hold a description, category, labels, due date, time records,
checklist items, images, links, and audio recordings.

## Assign a task agent

A task can have one task agent. Assigning one gives that task a persistent AI
assistant with its own inference setup, report, wake state, and proposal
history. The agent reads the task and its linked context, distils the current
situation into a short report, and can surface proposed task or checklist
changes for you to confirm or reject.

The assigned agent appears below the task's checklists and linked tasks. Its
collapsed card keeps the latest summary, automatic-update control, **Wake
agent** action, and active model route visible without making the task page
feel like an AI console.

Choose **Read more** to expand the complete report in place. The expanded card

### Choose automatic or manual updates

New task agents start with **Automatic updates off**. Assignment alone does not
spend tokens in the background. While updates are off, Lotti continues to
notice relevant changes to the task and its linked entries, marks an older
summary as out of date, and waits for you to choose **Wake agent**. A manual
wake remains available whenever the agent has a working AI setup.

<ManualScreenshot
  caseId="tasks/agent-card-manual"
  alt="Habitat Watcher task-agent card with automatic updates off, an out-of-date report notice, and a manual Wake agent action"
  caption="Manual mode observes changes and reports staleness without starting inference."
/>

When **Automatic updates** are on, Lotti bundles matching task changes for
about two minutes before scheduling a refresh. This coalescing avoids an agent
wake for every small edit. **Wake agent** is still available for an immediate,
user-requested refresh.

Turning automatic updates off clears its pending countdown and queued
automatic refreshes, but does not cancel a wake you explicitly requested.
Turning it on again applies to future changes; it does not replay changes that
arrived while automation was off.

## Priority

Lotti uses four compact priority levels:

| Priority | Meaning | Use it for |
| --- | --- | --- |
| P0 | Urgent | Work that genuinely needs attention next |
| P1 | High | Important work that should happen soon |
| P2 | Medium | The normal default |
| P3 | Low | Work that can wait |

Open a task and use the priority control in its header to change the level.
Existing tasks without an explicit priority behave as P2.

## Filter and order the task list

Open the task filters and select one or more priorities. Clear that selection to
return to all priorities. Priority is one of several filters that can be
combined with categories and other task properties.

<ManualScreenshot
  caseId="tasks/filters"
  alt="Task filter sheet with sort, status, priority, category, and label controls"
/>

The default priority order is P0 through P3. Within a priority, newer tasks
appear first unless another active view or sort changes the result.

## Use checklists for the next layer down

A checklist item should be smaller and more concrete than its task. “Publish
the manual” is a task; “verify every navigation link” is a checklist item.
Checklists can be edited directly and may also be proposed by configured AI
skills from task context.

## Add task context

Use the task's **Add** action to attach a checklist, another task, an audio
recording, a timer, a text entry, or an image without leaving the task. The
available actions depend on the current platform and context.

each entry type.

## Generate cover art from real task imagery

If an image-generation skill is configured, open the task's generation menu
and choose its cover-art action. Lotti first shows images already attached to
the task, plus available cover images from linked tasks. Select up to five
references to guide the result, or continue without a reference.

<ManualScreenshot
  caseId="tasks/cover-references"
  alt="Generate Cover Art sheet showing five distinct Project Waddle reference images from the task context"
  caption="The picker uses the same task imagery visible throughout the Project Waddle demo."
/>

After you continue, image generation runs in the background. The progress
view uses Lotti's production thinking visualizer and can be closed without
cancelling the job.

<ManualScreenshot

The generated image becomes the task cover and follows the task into list,
detail, project, and Daily OS views. A strong cover should remain recognizable
when the wide source image is cropped into a compact task thumbnail.

## Link related tasks

Open the task's **Linked** section and choose to link an existing task. Search
the picker, then select the related outcome. Lotti excludes the current task
and tasks that are already linked in that direction.

<ManualScreenshot
  caseId="tasks/link-picker"
  alt="Production task-link picker listing related Project Waddle tasks and their status, priority, category, and due-date context"
/>

Task links are directional: the current task points to the task you choose.
Both sides remain reachable through their linked-entry sections, while each
task keeps its own status, schedule, and attachments.

## Use voice without losing control

Record a note inside the task when it is faster to explain the situation aloud.
With transcription and task skills configured, Lotti can turn that context into
text and proposed checklist changes. Review the result, especially names,
dates, and destructive changes. See [Record audio](../plan-and-capture/recordings.mdx)
for the recording controls and processing options.

:::tip

Use P0 sparingly. If everything is urgent, the label stops carrying information.

:::
