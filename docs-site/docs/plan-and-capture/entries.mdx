# Add entries and context

Lotti uses one **Add** menu for the things you capture during the day. Open it
from the journal for a standalone entry, or from a task to keep the new entry
attached to that task.

## Choose the entry that matches the material

| Action | Use it for |
| --- | --- |
| **Checklist** | A short set of concrete steps inside the current task |
| **Task** | A separate outcome that needs its own status and context |
| **Audio Recording** | A voice note that can be replayed or transcribed |
| **Timer** | A timed work session linked to the current context |
| **Text Entry** | Notes, decisions, observations, or pasted information |
| **Import Image** | An existing image from the device |
| **Screenshot** | A new desktop screenshot attached as an image entry |

The menu is contextual. **Checklist** is available when a task can own the
items, while device-specific actions such as **Screenshot** only appear where
the platform supports them.

## Keep context with the work

When you open **Add** from a task, Lotti links the new entry back to that task.
That keeps recordings, notes, images, and time records together on the task
detail page. Use a standalone journal entry when the material does not belong
to a specific outcome yet; it can be linked later.

See [Record audio](./recordings.mdx) for capture and transcription, or
[Work with tasks](../organize-and-reflect/tasks.mdx) for task structure, cover
art, and task links.
