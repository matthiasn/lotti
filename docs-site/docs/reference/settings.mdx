# Find settings

Settings are grouped by the capability they control. Desktop uses a persistent
tree with the selected page beside it; mobile uses a focused list and opens one
settings page at a time.

Choose **Manual** in the final section of Settings to open this guide
in your default browser. On macOS, **Help → Manual** provides the same link.
The guide follows your system language by default; choose a different language
in **Settings → Advanced Settings → Language** when needed.

Common destinations include:

- **[Appearance](./appearance.mdx)** for light, dark, and automatic display
  modes plus independent theme palettes.
- **[Recording Style](./recording-style.mdx)** for the energy-orb or analogue
  meter shown while recording.
- **[Speech](./speech.mdx)** for the on-device voice, model, and speed used to
  read task summaries aloud.
- **[Keyboard shortcuts](./keyboard-shortcuts.mdx)** for the searchable desktop
  command catalog.
- **[Completion celebrations](./completion-celebrations.mdx)** for animations,
  haptics, styles, and their live playground.
- **[Advanced Settings](./advanced-settings.mdx)** for feature flags, logging,
  maintenance actions, onboarding diagnostics, and build information.
- **Categories** for planning boundaries and **Labels** for cross-cutting tags.
- **Habits** and **Dashboards** for reflection structures.
- **AI settings** for providers, models, profiles, and skills.
- **Daily OS** for the planner's inference profile and greeting.
- **Sync** for connection and activity state.

The exact set can vary with platform capabilities and enabled features.

## Manage definitions

Open **Settings → Definitions** to manage the structures that organize your
data. Categories set planning and privacy boundaries, labels tag tasks across
categories, habits define recurring routines, dashboards arrange reflection
views, and measurables define the numbers you track over time.
