# Browse What's New

What's New presents release notes inside Lotti, matched to the version you are
running. It can open automatically after an eligible first launch or version
change, and it is also available from the **What's New** entry in Settings when
that capability is enabled.

Release notes are ordered newest first. Each page combines a release-specific
hero image, its exact app version, and a short account of the user-visible
changes.

## Read the current release

The version badge identifies the release whose notes are on screen. **NEW**
marks the newest release in the current set; it does not mean every item on the
page is unfinished or experimental.

The body scrolls independently when the release is longer than the available
height. Links use the normal Lotti link handler, and images are preloaded when
available so moving between releases stays smooth.

## Move through release history

Use the footer controls to navigate:

- the right chevron opens the next older release;
- the left chevron returns to a newer release;
- the position markers show where the current page sits in the loaded set;
- **Skip** closes a multi-release set before the last page and marks the whole
  loaded set as seen; and
- **Done** appears on the oldest page and also marks the whole set as seen.

Closing the modal by dragging it away or selecting the dimmed area around it is
more conservative: Lotti marks only the releases you actually reached, through
the furthest page viewed. Releases you did not reach can therefore appear the
next time What's New opens.

## Revisit notes you already dismissed

When no unseen releases remain, opening What's New shows a **You're all caught
up** state. Choose **View past releases** to clear the local seen markers and
browse the available release set again. This changes only the local release-note
history; it does not alter the installed app version or download another build.

What's New never shows notes for a release newer than the installed Lotti
version. If release content cannot be loaded, the app falls back to an empty
state rather than presenting notes for the wrong build.

For the exact build number, platform, local entry totals, and task counts, see
[About Lotti](./advanced-settings.mdx#verify-the-build-and-local-totals).
