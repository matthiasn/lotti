# Spouštění a sledování agentů

Agenti umožňují Lotti plnit vymezenou roli přes více cyklů probuzení.
Každá běžící instance pochází ze šablony agenta, používá nastavení inference a
uchovává vlastní stav, zprávy, pozorování a historii aktivity. Celý systém
zobrazíš v **Nastavení → Agenti**.

Zprávy, pozorování a probuzení ve frontě popisují pracovní stav agenta;
nepřepisují potají tvůj osobní deník. Když agent úkolu chce změnit data úkolu,
Lotti ti předloží návrh ke kontrole. Ovládání schvalování pro jednotlivé úkoly
a úzkou výjimku pro první název a jazyk najdeš v části
[Kontroluj navržené změny](../organize-and-reflect/tasks.mdx#kontroluj-navržené-změny).

Spotřeba tokenů agentů se vykazuje společně s veškerou další AI prací v části [Kontrola využití a dopadu AI](./usage.mdx).

## Kontroluj běžící instance

V **Instancích** najdeš každého agenta a relaci vývoje. Řádky ukazují druh
agenta, životní cyklus, přiřazenou šablonu a duši. Seskupení podle duše se hodí,
když více odpovědností sdílí jeden hlas a rozhodovací styl.

Zbývající karty instance uchovávají různé podklady:

- **Zprávy** obsahují uložené snímky zpráv agenta.
- **Konverzace** seskupují zprávy podle cyklu probuzení.
- **Pozorování** ukazují zachycená zjištění bez míchání do zpráv.
- **Aktivita** ukazuje chronologickou stopu zpráv a nástrojů.

**Pozastavit** uchová instanci, ale zastaví běžnou práci. **Znovu analyzovat**
požádá agenta úkolu, aby znovu posoudil aktuální předmět. **Zničit** je
destruktivní a používej ho jen tehdy, když už nechceš samotnou instanci; změna
šablony pro budoucí agenty nevyžaduje ničit nesouvisející instance.

## Rozuměj cyklům probuzení

V **Cyklech probuzení** uvidíš práci probíhající nyní i práci naplánovanou na
později. **Čekající** probuzení vyžádal agent. **Naplánované** vložil systém nebo
plánovač. Každý řádek drží propojený úkol, projekt nebo den vedle odpovědného
agenta a živého odpočtu.

Smazání probuzení z fronty odstraní jeho časovač; nesmaže agenta ani propojený
úkol. Pokud se probuzení okamžitě vrátí, zkontroluj stav agenta nebo
automatizaci, která ho vyžádala, místo opakovaného mazání příznaku.

Pokračuj na [Šablony agentů a duše](./agent-blueprints.mdx), kde určíš, co
instance dělají a jak komunikují.
