# Přiřaď agenta úkolu

Úkol může mít jednoho agenta. Přiřazením získá trvalého AI asistenta s vlastním
nastavením inference, zprávou, stavem probuzení a historií návrhů. Agent čte
úkol a jeho propojený kontext, shrne aktuální situaci do krátké zprávy a může
předložit změny úkolu nebo kontrolního seznamu, které potvrdíš či odmítneš.
Souhrnný pohled napříč všemi běžícími agenty najdeš v části [Spouštění a
sledování agentů](../ai-and-automation/agents.mdx).

Přiřazený agent se zobrazí pod kontrolními seznamy a propojenými úkoly. Jeho
sbalená karta udržuje viditelné nejnovější shrnutí, ovládání automatických
aktualizací, akci **Probudit agenta** a aktivní trasu modelu, aniž by ze stránky
úkolu dělala konzoli AI.

**Otevřít interní informace agenta**, pokud potřebuješ zprávy, konverzace,
pozorování nebo podrobnosti běhu.

## Kontroluj navržené změny

Dokončené probuzení může aktualizovat zprávu a přidat návrhy změn. Automatické
aktualizace rozhodují, *kdy se agent probudí*; nedávají mu oprávnění upravovat
úkol. Návrhy čekají, dokud o nich nerozhodneš.

<ManualScreenshot
  caseId="tasks/agent-suggestions"
  alt="Karta agenta Habitat Watcher se dvěma čekajícími návrhy Project Waddle, jednotlivým odmítnutím a potvrzením a akcí Potvrdit vše"
  caption="Agent navrhuje položku kontrolního seznamu pro test krmítka a upravený odhad; žádná změna ještě nebyla použita."
/>

Křížkem návrh odmítneš; zaškrtnutím ho potvrdíš a použiješ. Návrh můžeš také
tažením doleva odmítnout nebo doprava potvrdit. Když čeká několik návrhů,
akce **Potvrdit vše** použije aktuálně uvedené změny společně.
Hromadné návrhy kontroluj pečlivě, zejména změny stavu, data a kontrolního
seznamu.

V **Historii** se můžeš vrátit k nedávným výsledkům. Rozlišuje potvrzené změny
od odmítnutých. Odmítnutá položka může znamenat tvoje rozhodnutí nebo návrh,
který agent stáhl poté, co ho novější kontext úkolu učinil zastaralým.

Ruční **Probudit agenta** může vytvořit stejné zprávy a návrhy jako automatické
probuzení. Vypnutí automatických aktualizací zastaví budoucí probouzení na
pozadí; nepoužije, nezahodí ani jinak nevyřeší návrhy, které už čekají na tvoje
rozhodnutí.

## Vyber automatické nebo ruční aktualizace

Noví agenti úkolů začínají s vypnutými **Automatickými aktualizacemi**. Samotné
přiřazení nespotřebovává tokeny na pozadí. Když jsou aktualizace vypnuté, Lotti
dál sleduje relevantní změny úkolu a propojených záznamů, označí starší shrnutí
za zastaralé a čeká na volbu **Probudit agenta**. Ruční probuzení je dostupné,
kdykoli má agent funkční nastavení AI.

<ManualScreenshot
  caseId="tasks/agent-card-manual"
  alt="Karta agenta Habitat Watcher s vypnutými automatickými aktualizacemi, upozorněním na zastaralou zprávu a ruční akcí Probudit agenta"
  caption="Ruční režim sleduje změny a hlásí zastarání, aniž by spustil inferenci."
/>

Když jsou **Automatické aktualizace** zapnuté, Lotti přibližně dvě minuty
seskupuje odpovídající změny úkolu a teprve potom naplánuje obnovení. Tím se
vyhne probuzení agenta po každé drobné úpravě. **Probudit agenta** zůstává
dostupné pro okamžité, uživatelem vyžádané obnovení.

Vypnutí automatických aktualizací zruší čekající odpočet a automatická
obnovení ve frontě, ale nezruší probuzení, které jsi výslovně vyžádal. Opětovné
zapnutí platí pro budoucí změny; nepřehraje změny, které přišly během vypnuté
automatizace.
