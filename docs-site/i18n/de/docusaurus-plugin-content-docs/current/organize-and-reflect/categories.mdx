# Mit Kategorien organisieren

Eine Kategorie ist das primäre strukturelle Zuhause eines Eintrags. Verwende
sie, wenn ein Arbeitsbereich eigene Planungsregeln, Datenschutzgrenzen,
visuelle Identität
oder KI-Kontext braucht. Im Demo-Arbeitsbereich trennen **Pinguinbetrieb**,
**Missionskontrolle**, **Fischdiplomatie** und **Menschenwartung** tatsächlich
verschiedene Expeditionsbereiche.

## Eine Kategorie erstellen

1. Öffne **Einstellungen → Kategorien**.
2. Wähle **Kategorie erstellen**.
3. Gib einen eindeutigen Namen ein und wähle Farbe sowie Symbol.
4. Erstelle die Kategorie und öffne sie erneut, um ihr Verhalten einzustellen.

Beim Erstellen geht es bewusst nur um die Identität. Planung, Datenschutz,
Sprache und Automatisierung stehen im vollständigen Editor bereit, sobald die
Kategorie existiert.

## Das Verhalten konfigurieren

<ManualScreenshot

/>

Der Editor steuert mehr als das Aussehen:

- **Favorit** macht eine wichtige Kategorie leichter erkennbar und auswählbar.
- **Privat** verbirgt ihren Inhalt, solange private Einträge ausgeblendet sind.
  Das ist eine Sichtbarkeitsgrenze, keine getrennte Verschlüsselung der
  Kategorie selbst.
- **Aktiv** bestimmt, ob sie für neue Einträge gewählt werden kann. Deaktivieren
  bewahrt vorhandene Daten.
- **Tagesplanung** lässt die Kategorie an der Daily-OS-Planung teilnehmen.
- **Sprache** ist die Standardsprache neuer Aufgaben dieser Kategorie.

Aktiviere **Tagesplanung** gezielt. Mit jeder Kategorie wird der Planer-Kontext
breiter und das tägliche Ritual unruhiger.

## Kategoriespezifischen Automatisierungskontext ergänzen

Eine Kategorie kann Standards für neue Aufgaben liefern: ein Inferenzprofil
und eine Agentenvorlage. Das sind Vorgaben, keine dauerhaften Sperren;
Aufgabeneinstellungen können später abweichen.

Das Wörterbuch für **Spracherkennung** hilft bei Namen, Orten, Abkürzungen und
Fachwortschatz. Pinguinbetrieb bringt Lotti zum Beispiel `Project Waddle`,
`Sir Flatterviel`, `Sardine` und `Europa` bei.

Wenn du erzeugte Checklisten-Titel manuell korrigierst, kann Lotti Vorher-
Nachher-Beispiele für künftige KI-Vorschläge behalten. Prüfe sie hier und
entferne Korrekturen, die spätere Ergebnisse nicht beeinflussen sollen.

## Kategorien oder Labels

Verwende die Kategorie für den primären Ort der Arbeit und ein
[Label](./labels.mdx) für eine übergreifende Eigenschaft. **Pinguinbetrieb** ist
eine Kategorie; **Habitat kritisch** ist ein Label, das auch auf Arbeit der
Missionskontrolle passen kann.

Halte dein Kategoriensystem klein genug für eine offensichtliche Wahl. Ergänze
eine Kategorie, wenn sie Planung, Filter, Datenschutz oder Automatisierung
ändert – nicht nur, weil ein neues Substantiv aufgetaucht ist.
