# Arbeit mit Labels markieren

Labels ergänzen eine zweite, flexible Organisationsebene. Eine Aufgabe hat
eine primäre Kategorie, kann aber mehrere Labels für Initiativen, Dringlichkeit,
Wartezustände oder andere Eigenschaften über Kategoriegrenzen hinweg tragen.

Intergalaktische Pinguinlogistik verwendet Labels bewusst:

- **Project Waddle** verbindet Startarbeit über Pinguinbetrieb,
  Missionskontrolle und Fischdiplomatie hinweg.
- **Habitat kritisch** markiert Arbeit, die vor dem Einzug der Expedition ins
  Orbitalhabitat fertig sein muss.
- **Wartet auf Missionskontrolle** beschreibt einen echten Blocker.
- **Sardinenmarkt sensibel** hält private Verhandlungen erkennbar.

## Ein Label erstellen und bearbeiten

1. Öffne **Einstellungen → Labels**.
2. Wähle **Label erstellen**.
3. Gib einen knappen Namen und optional eine Beschreibung ein.
4. Wähle eine Farbe, die neben anderen Labels unterscheidbar bleibt.
5. Entscheide über Datenschutz und zulässige Kategorien.
6. Speichere.

Eine Beschreibung lohnt sich, wenn ein Label mehrdeutig sein könnte. Sie gibt
ihm eine operative Bedeutung, ohne diese Erklärung in jeden Aufgabentitel zu
zwingen.

**Anwendbare Kategorien** begrenzen, wo das Label gewählt werden kann. Project
Waddle ist in den drei am Start beteiligten Expeditionsbereichen verfügbar; ein
Speziallabel kann auf eine Kategorie beschränkt bleiben. Auswahlfelder bleiben
so relevant, statt überall jedes Label anzubieten.

## Labels auf Aufgaben anwenden

Öffne eine Aufgabe und füge über ihre Label-Steuerung Labels hinzu oder entferne
sie. Labels arbeiten mit Status-, Prioritäts- und Kategorienfiltern zusammen.
Du kannst so etwa „offene P1-Aufgaben mit Habitat kritisch“ anzeigen, ohne für
diese temporäre Sicht eine neue Kategorie zu erfinden.

Wenn automatische Label-Vorschläge aktiv sind, gilt das Entfernen als bewusste
Ablehnung. Lotti merkt sich diese Entscheidung, damit derselbe Vorschlag nicht
sofort ohne neuen Grund zurückkehrt.

## Labels nützlich halten

- Bevorzuge Labels, die eine Entscheidung, einen Filter oder eine Prüfung ändern.
- Verwende ein stabiles Label statt mehrerer Schreibvarianten.
- Prüfe Nutzungszahlen vor dem Löschen oder Zusammenführen.
- Nutze Datenschutz, wenn bereits der Labelname sensiblen Kontext verrät.
- Räume Labels auf, die keine aktive Arbeit mehr beschreiben.

Ist etwas der primäre, dauerhafte Ort der Arbeit, verwende stattdessen eine
[Kategorie](./categories.mdx).
