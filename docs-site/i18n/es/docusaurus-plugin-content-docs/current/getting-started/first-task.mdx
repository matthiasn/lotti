# Crea tu primera tarea

Una tarea reúne un resultado, su lista de verificación y las notas, grabaciones, imágenes o registros de tiempo que explican el trabajo.

## Empieza por el camino más rápido

Si sigues en el onboarding, di el resultado o elige **¿Prefieres escribir?**. Escoge el área a la que pertenece la tarea; Lotti usa esa categoría para mantenerla visible en las vistas adecuadas y seleccionar las automatizaciones configuradas para ella.

## Crea una tarea desde la lista de tareas

1. Abre **Tareas**.
2. Usa la acción **+** y elige **Tarea**.
3. Escribe un título orientado al resultado.
4. Elige una categoría cuando corresponda claramente.
5. Guarda la tarea.

El espacio de trabajo de tareas mantiene juntas la lista y la tarea seleccionada en escritorio. En móvil, al seleccionar una tarea se abre la vista detallada a pantalla completa. En ambos diseños, la portada de la tarea y el contexto de la categoría facilitan recorrer una lista cargada.

«Publicar el manual de la V1» es más útil que «Documentación», porque el título te dice cómo será el resultado terminado.

## Añade contexto útil

Abre la tarea y usa su acción para añadir una nota, grabación, imagen u otra entrada. Conserva el material con la tarea cuando vaya a ayudarte a ti —o a un asistente de IA— a entender después cuál es la siguiente acción.

## Haz que se pueda llevar a cabo

Añade una lista de verificación breve con pasos concretos. Si hay un proveedor de IA y las habilidades pertinentes configurados, Lotti también puede convertir una grabación o un contexto escrito en sugerencias para la lista. Revisa los cambios propuestos antes de aplicarlos.

## Decide cuándo importa

- Establece una fecha límite cuando haya un plazo real.
- Establece la prioridad para expresar urgencia.
- Lleva la tarea a un check-in de Daily OS cuando pertenezca al plan de hoy.

Continúa con [gestión de tareas](../organize-and-reflect/tasks.mdx) para conocer la prioridad, los filtros y los flujos asistidos por voz.
