## Los elementos básicos, sin fingir que son todos iguales

| Elemento básico | Para qué usarlo | Ejemplo |
| --- | --- | --- |
| Categoría | Un área significativa de la vida o el trabajo | Salud, familia, investigación |
| Tarea | Un resultado que necesita avanzar | Preparar el lanzamiento de la V1 |
| Lista de verificación | Pasos concretos dentro de una tarea | Comprobar las notas de la versión |
| Entrada | Contexto o prueba | Nota, imagen, grabación de audio |
| Bloque planificado | Un espacio propuesto en el día | Revisión de lanzamiento, 10:00–11:00 |
| Registro de tiempo | Lo que ocurrió realmente | Revisión de lanzamiento, 10:18–11:07 |
| Hábito | Un comportamiento recurrente que evalúas | Tomar la medicación |
| Valor medible | Una observación numérica | Frecuencia cardíaca en reposo |
| Panel | Una vista de mediciones útiles | Resumen de entrenamiento |

## Los agentes son invitados, no copropietarios

Un agente puede leer el contexto que le das, formarse una opinión, resumir una tarea desordenada y proponer el siguiente cambio útil. Su informe es una opinión con procedencia, no un hecho nuevo en tu historial. Los cambios propuestos en tareas, listas de verificación, estado o fechas esperan a que los confirmes o descartes. Las excepciones acotadas son un título o idioma iniciales para una tarea que, por lo demás, está vacía; después se aplica la misma regla de aprobación humana.

No es cortesía añadida a un prompt. Lotti mantiene separados el diario personal y la memoria de trabajo del agente, de modo que el agente no puede tratar tu historial como una libreta de borradores. Puedes usar IA, ajustarla o prescindir de ella por completo.

## Una primera órbita práctica

1. Crea categorías que realmente uses para tomar decisiones.
2. Captura el trabajo como tareas y conserva el material de apoyo adjunto.
3. Usa Daily OS para decidir qué pertenece a un día concreto.
4. Registra el tiempo y las observaciones reales sin obligarlos a coincidir con el plan.
5. Revisa hábitos y paneles para encontrar patrones, no para buscar días individuales perfectos.

## Mantén clara la propiedad

- Una tarea vinculada es dueña de su título y su categoría.
- Un bloque de Daily OS es dueño de su ubicación y duración.
- Una entrada de tiempo es dueña del registro de lo que ocurrió.
- La IA propone; tú revisas y decides.

:::tip El objetivo no es una cronología impecable

Si el plan cambió, deja que cambie. Si el día se torció, registra lo que sí ocurrió. Un sistema útil te ayuda a ver la forma de tu vida; no te pide que representes una versión más productiva de ella para la base de datos.

:::
