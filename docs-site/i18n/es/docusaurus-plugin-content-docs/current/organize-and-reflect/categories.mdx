# Organiza con categorías

Una categoría es el principal contenedor estructural de una entrada. Úsala cuando un conjunto de trabajo necesite sus propias reglas de planificación, ámbito de privacidad, identidad visual o contexto de IA. En el espacio de trabajo de demostración, **Operaciones de pingüinos**, **Control de misión**, **Diplomacia de los peces** y **Mantenimiento humano** separan áreas realmente distintas de la expedición.

## Crea una categoría

<TutorialVideo
  scenario="category_setup"
  title="Configura una categoría"
  caption="Recorrido narrado: crear una categoría, configurar sus opciones de favorito/privacidad y sus valores predeterminados de IA."
/>

1. Abre **Ajustes → Categorías**.
2. Selecciona **Crear categoría**.
3. Introduce un nombre, color e icono distintivos.
4. Créala y vuelve a abrirla para configurar su comportamiento.

La creación empieza intencionadamente con los campos de identificación. Las opciones de planificación, privacidad, idioma y automatización están en el editor completo después de que exista la categoría.

## Configura el comportamiento de una categoría

<ManualScreenshot
  caseId="categories/detail"

/>

El editor controla más que la apariencia:

- **Favorita** hace que una categoría importante sea más fácil de reconocer y seleccionar.
- **Privada** oculta su contenido salvo cuando se muestran las entradas privadas. Es un límite de visibilidad, no un cifrado independiente para la categoría.
- **Activa** controla si se puede seleccionar la categoría para nuevas entradas. Desactivarla conserva los datos existentes.
- **Plan del día** permite que la categoría participe en la planificación de Daily OS.
- **Idioma** proporciona el idioma predeterminado para nuevas tareas de la categoría.

Usa **Plan del día** de forma selectiva. Activarlo para todas las categorías amplía el contexto del planificador y hace que el ritual diario tenga más ruido.

## Añade contexto de automatización específico de la categoría

Una categoría puede proporcionar valores predeterminados para nuevas tareas: un perfil de inferencia y una plantilla de agente. Son valores predeterminados, no bloqueos permanentes; las opciones de cada tarea aún pueden cambiarse después.

El diccionario de **Reconocimiento de voz** es útil para nombres, lugares, acrónimos y vocabulario de dominio que la transcripción suele pasar por alto. Operaciones de pingüinos, por ejemplo, enseña a Lotti `Project Waddle`, `Sir Flaps-a-Lot`, `sardine` y `Europa`.

Cuando corriges manualmente títulos de listas de verificación generados, Lotti puede conservar ejemplos del antes y el después para futuras sugerencias de IA. Revisa aquí esos ejemplos y elimina cualquier corrección que no deba orientar resultados posteriores.

## Categorías frente a etiquetas

Usa una categoría como hogar principal del trabajo y una [etiqueta](./labels.mdx) para una propiedad transversal. **Operaciones de pingüinos** es una categoría; **Hábitat crítico** es una etiqueta que también puede aplicarse al trabajo de Control de misión.

Mantén el sistema de categorías lo bastante pequeño para que elegir una sea obvio. Añade una categoría cuando cambie la planificación, el filtrado, la privacidad o la automatización, no solo porque haya aparecido un sustantivo nuevo.
