# Gestiona el trabajo como proyectos

Los proyectos se sitúan entre las categorías y las tareas. Una categoría como **Operaciones de pingüinos** define un área amplia de responsabilidad; **Project Waddle** agrupa las tareas concretas necesarias para preparar el hábitat orbital de pingüinos.

## Encuentra el proyecto que necesitas

La búsqueda coincide con títulos de proyectos, texto de proyecto almacenado y nombres de categorías. El control de filtro limita la lista por estado o categoría del proyecto. Un filtro permanece inactivo hasta que selecciones **Aplicar**.

<ManualScreenshot
  caseId="projects/filters"
  alt="Diálogo de filtro de proyectos con controles de estado y categoría sobre la lista de proyectos de Logística Interplanetaria de Pingüinos"
/>

Los proyectos permanecen agrupados por categoría después de filtrar. Así es más fácil distinguir iniciativas con nombres similares y cada área operativa sigue visible.

## Crea un proyecto

Selecciona el botón **+** en Proyectos e introduce:

- un título obligatorio;
- una categoría opcional; y
- una fecha objetivo opcional.

<ManualScreenshot
  caseId="projects/create"
  alt="Formulario Crear proyecto para Establecer la reserva lunar de sardinas en la categoría Operaciones de pingüinos"
  caption="El formulario de creación es una hoja inferior en el móvil y un diálogo centrado en el escritorio."
/>

Un proyecto nuevo empieza **Abierto**. Si su categoría tiene una plantilla de agente de proyecto, Lotti también crea ese agente después de guardar el proyecto. El proyecto sigue funcionando normalmente cuando no existe una plantilla coincidente.

Usa un título que describa el resultado o la misión duradera. «Establecer la reserva lunar de sardinas» dice más que «Sardinas».

## Edita los ajustes del proyecto

La ruta de ajustes del proyecto ofrece el editor de formulario completo para el estado, título, fecha objetivo, salud, asignación de agente, cambios propuestos por el agente y tareas vinculadas. También mantiene las acciones **Cancelar** y **Guardar** fijadas abajo, de modo que los cambios pendientes del formulario sean explícitos.

## Comprende el estado del proyecto

| Estado | Úsalo cuando |
| --- | --- |
| Abierto | El proyecto existe, pero la ejecución activa aún no ha empezado. |
| Activo | El trabajo avanza en este momento y pertenece a la planificación diaria. |
| En observación | No hay trabajo programado, pero el resultado aún necesita observación. |
| En pausa | El trabajo está pausado por un motivo conocido. |
| Completado | Se ha alcanzado el resultado previsto. |
| Archivado | El proyecto se conserva como historial, pero ya no está operativo. |

Selecciona la píldora de estado en la página de detalle del proyecto para cambiarlo. Los cambios de estado, categoría y fecha objetivo se guardan inmediatamente; no hay un botón de guardar separado en esta superficie de detalle centrada en la lectura.

## Lee la salud del proyecto y su informe

<ManualScreenshot
  caseId="projects/detail"
  alt="Detalle de Project Waddle con una puntuación de salud de 82, cinco de ocho tareas completadas, un bloqueo y un informe de IA actualizado hace dos horas"
  caption="El escritorio mantiene la lista de proyectos y el detalle uno junto a otro. El móvil abre el mismo detalle como una página completa."
/>

El panel de salud resume la evaluación actual del agente del proyecto:

- la puntuación de salud es una señal compacta, no un sustituto del trabajo subyacente;
- el progreso de tareas muestra los recuentos de completadas y bloqueadas;
- el informe ofrece un resumen actual y puede ampliarse para ver su contenido completo; y
- las recomendaciones llaman la atención sobre los siguientes pasos concretos.

En Project Waddle, una puntuación de 82 y cinco tareas completadas son alentadoras, pero el alimentador de peces en gravedad cero sigue bloqueando el lanzamiento. Lee la explicación antes de tratar una puntuación como un veredicto.

Cuando hay configurado un agente de proyecto, el menú del informe puede solicitar un informe nuevo o cancelar una ejecución programada. Sin agente, el proyecto sigue mostrando sus tareas y cualquier texto de proyecto almacenado.

## Mantén el trabajo del proyecto en tareas

<ManualScreenshot
  caseId="projects/tasks"
  alt="Panel de tareas de Project Waddle que enumera la inspección del hábitat, la calibración del alimentador de peces en gravedad cero y la confirmación de cápsulas de carga de sardinas con estimaciones y estados"
/>

Las filas de tareas del proyecto muestran el título de cada tarea, un resumen breve, la estimación y el estado actual. Selecciona una fila para abrir la tarea. Los detalles de la tarea siguen siendo la referencia principal para las descripciones, grabaciones, listas de verificación, imágenes, registros de tiempo e ilustración de portada.

Asigna una tarea a un proyecto desde el control de proyecto en el encabezado de la tarea. Los proyectos tienen ámbito de categoría, así que asigna primero una categoría a la tarea antes de elegir un proyecto. Una tarea pertenece como máximo a un proyecto a la vez; asignar otro proyecto mueve el vínculo en lugar de duplicarlo.

## Proyectos, categorías y tareas

Usa los tres niveles de forma deliberada:

- **Categoría:** un área operativa duradera, como Operaciones de pingüinos.
- **Proyecto:** un resultado que requiere varias tareas, como Project Waddle.
- **Tarea:** una unidad ejecutable, como recalibrar el alimentador de peces en gravedad cero.

Evita crear un proyecto para una sola acción sencilla. Usa uno cuando varias tareas necesiten una fecha objetivo, estado, acumulado de progreso o informe operativo compartidos.
