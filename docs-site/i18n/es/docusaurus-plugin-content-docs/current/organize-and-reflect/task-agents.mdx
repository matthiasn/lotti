# Asigna un agente de tarea

Una tarea puede tener un agente de tarea. Al asignarlo, esa tarea obtiene un asistente de IA persistente con su propia configuración de inferencia, informe, estado de activación e historial de propuestas. El agente lee la tarea y su contexto vinculado, sintetiza la situación actual en un informe breve y puede mostrar cambios propuestos para la tarea o la lista de verificación que puedes confirmar o rechazar. Para tener una vista de todo el sistema con todos los agentes en ejecución, consulta [Ejecuta y supervisa agentes](../ai-and-automation/agents.mdx).

El agente asignado aparece debajo de las listas de verificación y las tareas vinculadas. Su tarjeta contraída mantiene visibles el último resumen, el control de actualizaciones automáticas, la acción **Ejecutar agente** y la ruta de modelo activa, sin convertir la página de tareas en una consola de IA.

## Revisa los cambios propuestos

Una ejecución terminada puede actualizar el informe y añadir cambios propuestos. Las actualizaciones automáticas deciden *cuándo se ejecuta el agente*; no le conceden permiso para editar la tarea. Las propuestas siguen pendientes hasta que actúes sobre ellas.

<ManualScreenshot
  caseId="tasks/agent-suggestions"
  alt="Tarjeta del agente Vigilante del hábitat con dos propuestas pendientes de Project Waddle, acciones individuales para descartar y confirmar, y una acción Confirmar todo"
  caption="El agente propone un elemento de lista de verificación para probar el comedero y una estimación revisada; todavía no se ha aplicado ninguno de los dos cambios."
/>

Usa la cruz para **Descartar** una propuesta o la marca de verificación para **Confirmarla** y aplicarla. También puedes deslizar una propuesta a la izquierda para descartarla o a la derecha para confirmarla. Cuando hay varias propuestas pendientes, **Confirmar todo** aplica juntas las modificaciones que se muestran. Revisa las propuestas en bloque con cuidado, sobre todo los cambios de estado, fecha y lista de verificación.

Abre **Historial** para volver a consultar resultados recientes. Distingue los cambios confirmados de los descartados. Una entrada descartada puede representar tu decisión o una propuesta que el agente retiró porque un contexto más reciente de la tarea la dejó obsoleta.

La acción manual **Ejecutar agente** puede generar los mismos tipos de informe y propuestas que una ejecución automática. Desactivar las actualizaciones automáticas detiene futuras ejecuciones en segundo plano; no aplica, descarta ni resuelve de ningún otro modo las propuestas que ya esperan tu decisión.

## Elige actualizaciones automáticas o manuales

Los nuevos agentes de tarea empiezan con **Actualizaciones automáticas desactivadas**. La asignación por sí sola no consume tokens en segundo plano. Mientras están desactivadas, Lotti sigue detectando cambios relevantes en la tarea y sus entradas vinculadas, marca un resumen anterior como desactualizado y espera a que elijas **Ejecutar agente**. La ejecución manual sigue disponible siempre que el agente tenga una configuración de IA funcional.

<ManualScreenshot
  caseId="tasks/agent-card-manual"
  alt="Tarjeta del agente Vigilante del hábitat con las actualizaciones automáticas desactivadas, un aviso de informe desactualizado y una acción manual Ejecutar agente"
  caption="El modo manual observa los cambios e informa de que el resumen está desactualizado sin iniciar una inferencia."
/>

Cuando las **Actualizaciones automáticas** están activadas, Lotti agrupa los cambios coincidentes de la tarea durante unos dos minutos antes de programar una actualización. Esta agrupación evita ejecutar al agente por cada pequeña edición. **Ejecutar agente** sigue disponible para una actualización inmediata solicitada por ti.

Desactivar las actualizaciones automáticas borra su cuenta atrás pendiente y las actualizaciones automáticas en cola, pero no cancela una ejecución que hayas solicitado explícitamente. Al volver a activarlas, se aplican a cambios futuros; no reproducen los cambios que llegaron mientras la automatización estaba desactivada.
