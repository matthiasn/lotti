Lotti utilizza un menu **Add** per le cose che catturate durante il giorno.

## Scegli la voce che corrisponde al materiale

| Azione | Usalo per |
| --- | --- |
| **Checklist** | Una breve serie di passi concreti all'interno dell'attività corrente |
| **Task** | Un risultato separato che ha bisogno del proprio stato e contesto |
| **Registrazione audio** | Una nota vocale che può essere riprodotta o trascritta |
| **Timer** | Una sessione di lavoro a tempo collegata al contesto corrente |
| **Voce di testo** | Note, decisioni, osservazioni o informazioni incollate |
| **Importa immagine** | Un'immagine esistente dal dispositivo |
| **Screenshot** | Un nuovo screenshot del desktop allegato come voce di immagine |

Il menu è contestuale. **Checklist** è disponibile quando un'attività può
contenere elementi, mentre le azioni specifiche del dispositivo, come
**Screenshot**, appaiono solo dove la piattaforma le supporta.

## Tenere il contesto con il lavoro

Quando si apre **Add** da un compito, Lotti collega la nuova voce di nuovo a quel compito.
Questo mantiene le registrazioni, le note, le immagini e i record di tempo insieme sul compito
pagina dei dettagli. Utilizzare una voce di giornale standalone quando il materiale non appartiene
a un risultato specifico ancora; può essere collegato in seguito.

Vedere [Record audio](./recordings.mdx) per la cattura e la trascrizione, o
[Lavorare con compiti](../organize-and-reflect/tasks.mdx) per la struttura delle mansioni, coprire
arte, e link di attività.
