# Revise o uso e o impacto da IA

Abra **Configurações → Configurações de IA → Uso e impacto** para entender
como os modelos de nuvem configurados estão sendo usados. A mesma análise
também pode ser acessada pela rota do painel de impacto de IA.

## Leia os cartões de resumo

Os cartões superiores resumem o período selecionado:

- **Custo** estima os gastos do provedor.
- **Energia** estima a eletricidade atribuível à inferência medida na nuvem.
- **CO₂e** expressa a estimativa de emissões associadas.
- **Tokens** conta tokens de modelo processados.
- **Solicitações** conta chamadas de inferência.

O indicador de comparação utiliza o período equivalente anterior quando ele
está disponível. Um aumento percentual é contexto, não automaticamente um
problema: uma semana de revisão de lançamento não deve ser comparada às cegas
com uma semana ociosa na colônia.

Energia, CO₂e e custo são medidos apenas para modelos de nuvem. A inferência
local pode consumir bastante energia do dispositivo sem aparecer nessas
estimativas de nuvem; portanto, esta página não é um medidor de energia
completo para o sistema inteiro.

## Alterar a análise

Use o controle de período e os botões de seta para percorrer o tempo ou pule
para **Este mês** e **Este ano**. Em seguida, escolha **Ambos**,
**Por categoria** ou **Por modelo**. Dentro de um gráfico, alterne entre
valores por dia, totais acumulados e participação, quando disponível.

As visualizações por categoria respondem “qual área de trabalho impulsionou o
uso?”; as visualizações por modelo respondem “qual modelo configurado
impulsionou o uso?”. Use ambas antes de alterar um perfil: um modelo de alto
custo pode ser justificado para uma categoria especializada, mas um
desperdício como padrão geral.

## Agir com base em um resultado surpreendente

Se o uso aumentar inesperadamente:

1. restringir o período ao primeiro dia afetado;
2. comparar categorias e modelos;
3. revisar a automação e a simultaneidade dos agentes;
4. inspecionar o perfil de inferência atribuído à categoria afetada; e
5. verificar o faturamento no provedor antes de tratar a estimativa do Lotti
   como uma fatura.

Para alterar a rota responsável pelas chamadas futuras, volte para
[Modelos e perfis de inferência](./models-and-profiles.mdx).
