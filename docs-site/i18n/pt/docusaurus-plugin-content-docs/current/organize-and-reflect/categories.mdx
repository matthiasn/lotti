# Organize com categorias

Uma categoria é o principal lar estrutural de uma entrada. Use uma quando um
conjunto de trabalho precisa de regras de planejamento próprias, limite de
privacidade, identidade visual ou contexto de IA. No espaço de trabalho de
demonstração, **Penguin Operations**, **Mission Control**, **Diplomacia dos
Peixes** e **Manutenção Humana** separam áreas genuinamente diferentes da
expedição.

## Crie uma categoria

1. Abra **Configurações → Categorias**.
2. Selecione **Criar categoria**.
3. Insira um nome, cor e ícone distintos.
4. Crie-a e depois abra a categoria novamente para configurar seu comportamento.

A criação começa intencionalmente pelos campos de identificação. As opções de
planejamento, privacidade, idioma e automação ficam no editor completo, depois
que a categoria existe.

O editor controla mais do que a aparência:

- **Favorito** torna uma categoria importante mais fácil de reconhecer e selecionar.
- **Privado** oculta o conteúdo dela, a menos que entradas privadas estejam
  sendo exibidas. É um limite de visibilidade, não uma criptografia separada
  para a categoria em si.
- **Ativo** controla se a categoria pode ser selecionada para novas entradas.
  Desativá-la preserva os dados existentes.
- **Planejamento diário** permite que a categoria participe do planejamento do
  Daily OS.
- **Idioma** fornece o idioma padrão para novas tarefas na categoria.

Use o **planejamento diário** com moderação. Ativá-lo para todas as categorias
amplia o contexto do planejador e torna o ritual diário mais ruidoso.

## Adicione contexto de automação específico da categoria

Uma categoria pode fornecer padrões para novas tarefas: um perfil de inferência
e um modelo de agente. São padrões, não travas permanentes; as escolhas no
nível da tarefa ainda podem ser alteradas depois.

O dicionário de **Reconhecimento de fala** é útil para nomes, lugares, siglas e
vocabulário de domínio que a transcrição costuma errar. A Penguin Operations,
por exemplo, ensina à Lotti `Project Waddle`, `Sir Flaps-a-Lot`, `sardine` e
`Europa`.

Quando você corrige manualmente os títulos de listas de verificação geradas, a
Lotti pode guardar os exemplos de antes e depois para futuras sugestões de IA.
Revise os exemplos aqui e remova qualquer correção que não deva orientar
resultados futuros.

## Categorias versus rótulos

Use uma categoria como o lar principal do trabalho e um [rótulo](./labels.mdx)
para uma propriedade transversal. **Operações Penguin** é uma categoria;
**Habitat crítico** é um rótulo que também pode ser aplicado ao trabalho do
Mission Control.

Mantenha o sistema de categorias pequeno o bastante para que escolher uma seja
óbvio. Adicione uma nova categoria quando ela mudar o planejamento, a
filtragem, a privacidade ou a automação — não apenas porque um substantivo novo
apareceu.
