# Rastreie hábitos e medições

Um hábito responde **este comportamento recorrente aconteceu?** Um mensurável
responde **que valor numérico eu observei?** Eles podem apoiar o mesmo objetivo
sem ser intercambiáveis: “Chamada do pinguim-imperador” é um hábito, enquanto
“Pinguins contabilizados” é um valor mensurável.

## Trabalhe com hábitos

A página Hábitos organiza as definições ativas entre o que está pendente agora,
o que está agendado para mais tarde e o que foi concluído. Seu resumo diário, o
histórico de consistência, as sequências e o gráfico de taxa de conclusão
ajudam você a perceber padrões sem tratar um dia perdido como a história
inteira.

### Crie e encontre definições de hábitos

1. Abra **Configurações → Hábitos**.
2. Selecione **Criar hábito**.
3. Dê ao comportamento um nome concreto e repetível e uma descrição útil.
4. Escolha sua categoria, opções e programação.
5. Salve.

  caption="Pesquise definições aqui. A estrela, o olho oculto e o cadeado identificam hábitos favoritos, inativos e privados."

<ManualScreenshot

O editor de definição inclui:

- **Categoria** para filtragem e contexto visual.
- **Painel** quando o hábito deve aparecer como um gráfico em um painel selecionado.
- **Favorito** para maior destaque.
- **Privado** para ocultá-lo, a menos que entradas privadas sejam mostradas.
- **Ativo** para controlar se ele aparece na página de Hábitos sem excluir
  seu histórico.

### Programe um hábito diário

<ManualScreenshot
  caseId="habits/schedule"
  alt="Programação da chamada do pinguim-imperador a partir de 1º de julho, exibida a partir das 06h com alerta às 06h30"
/>

O editor atual é voltado para hábitos diários. Defina uma **data de início**,
opcionalmente adie o horário a partir do qual o hábito é **mostrado a partir
de** e, opcionalmente, escolha quando a Lotti deve **mostrar um alerta**. O
modelo de dados também entende programações semanais e mensais, mas o editor
atual expõe os controles de tempo mais completos para hábitos diários.

### Registre o que realmente aconteceu

Abra um hábito pendente ou use sua ação rápida. O formulário de conclusão
detalhado aceita três resultados:

- **Sucesso** — o comportamento aconteceu.
- **Perdido** — era esperado e possível, mas não aconteceu.
- **Pular** — não se aplicava razoavelmente naquele dia.

<ManualScreenshot
  caseId="habits/record"
  alt="Formulário de preenchimento de hábito para inspecionar as vedações do habitat dos pinguins, com resultados de Sucesso, Pular e Perdido, além de data e notas"
/>

Use o estado que preserva a realidade. Um histórico sincero é mais útil do que
um gráfico de aparência perfeita. Os registros de conclusão são entradas do
diário, então visualizações posteriores podem reconstruir o resultado efetivo
de cada hábito e dia.

## Trabalhe com mensuráveis

Uma definição mensurável descreve o número antes de você registrá-lo: seu nome,
unidade, privacidade e como múltiplas observações se combinam nos gráficos.

<ManualScreenshot
  caseId="measurables/settings-list"
  alt="Configurações mensuráveis contendo pressão do habitat, pinguins contabilizados e sardinhas consumidas"
/>

### Crie um tipo de dados mensurável

1. Abra **Configurações → Tipos de dados mensuráveis**.
2. Selecione a ação de adicionar.
3. Adicione um nome, uma descrição opcional e uma abreviação concisa para a
   unidade.
4. Escolha a agregação padrão usada pelos gráficos.
5. Decida se ele deve ser favorito ou privado e salve.

<ManualScreenshot
  caseId="measurables/settings-detail"
  alt="Editor mensurável de pressão de habitat usando kPa e agregação média diária"
/>

Escolha a agregação com base no significado das leituras repetidas:

| Agregação | Use quando |
| --- | --- |
| Sem agregação | Cada observação individual é importante |
| Soma diária | As leituras se acumulam, como sardinhas consumidas |
| Máximo diário | O pico do dia é o sinal importante |
| Média diária | As leituras amostram um valor que varia, como a pressão do habitat |
| Soma horária | O acúmulo precisa de intervalos por hora mais finos |

A unidade é contexto de exibição, não um conversor. Se você definir a pressão
do habitat como `kPa`, registre os valores em kPa de forma consistente. Crie um
mensurável separado se uma unidade diferente representar uma série genuinamente
diferente.

### Registrar uma medição

Abra um painel que contenha o mensurável e selecione a ação de adicionar no
gráfico dele. Insira o valor observado; a Lotti mantém a unidade ao lado do
número, para que fique claro o que será salvo. Valores distintos recentes
aparecem como ações de **Log rápido** quando repetir um deles é útil.

<ManualScreenshot
  caseId="measurements/capture"
  alt="Editor de medição de sardinhas consumidas aberto a partir do gráfico preenchido do painel, com 84 sardinhas, sugestões de registro rápido, horário observado, comentário e ação Salvar"
/>

Use **Observado em** para o momento em que a leitura foi realmente feita, e não
a hora em que você a registra. A primeira página seleciona a data no
calendário. **Hoje** volta diretamente para a data atual.

<ManualScreenshot
  caseId="measurements/date"
  alt="Calendário de Observado em selecionando sexta-feira, 17 de julho de 2026, para a medição de sardinhas"
/>

Role até a seção de horário para definir a hora e os minutos. **Agora**
restaura o horário atual. Selecione **Concluído** para voltar ao rascunho da
medição, ainda preenchido, e depois **Salvar** para criar a entrada no diário.

<ManualScreenshot
  caseId="measurements/time"
  alt="Seletor de horário de Observado em ajustado para 10h30 para a medição de sardinhas"
/>

O comentário é opcional, mas é útil quando um número precisa de contexto
operacional — por exemplo, uma entrega, um turno incomum ou uma mudança no
método de medição. Salvar uma medição não altera a definição do mensurável nem
sua regra de agregação.

## Conecte os dois sem confundi-los

Uma chamada pode ser concluída com sucesso mesmo que apenas 36 pinguins tenham
sido contados — o hábito registra se o procedimento aconteceu, enquanto o
mensurável registra o resultado. Coloque os dois em um
[dashboard](./dashboards.mdx) quando a relação entre eles responder a uma
pergunta operacional recorrente.
