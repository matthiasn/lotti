# Execute o trabalho como projetos

Os projetos ficam entre as categorias e as tarefas. Uma categoria como
**Operações Penguin** define uma área ampla de responsabilidade; o **Project
Waddle** agrupa as tarefas concretas necessárias para preparar o habitat
orbital dos pinguins.

## Encontre o projeto que você precisa

A pesquisa procura nos títulos dos projetos, no texto armazenado do projeto e
nos nomes das categorias. O controle de filtro restringe a lista por status ou
categoria do projeto. Um filtro permanece inativo até você selecionar
**Aplicar**.

<ManualScreenshot
  caseId="projects/filters"
  alt="Caixa de diálogo de filtro de projeto com controles de status e categoria na lista de projetos da Intergalactic Penguin Logistics"
/>

Os projetos permanecem agrupados por categoria após a filtragem. Isso torna
mais fácil distinguir esforços com nomes parecidos e mantém cada área
operacional visível.

## Crie um projeto

Selecione o botão **+** em Projetos e informe:

- um título obrigatório;
- uma categoria opcional; e
- uma data prevista opcional.

<ManualScreenshot
  caseId="projects/create"
  alt="Formulário de criação de projeto para Estabelecer a reserva lunar de sardinhas na categoria Penguin Operations"
  caption="O formulário de criação é um painel inferior no celular e uma caixa de diálogo centralizada no desktop."
/>

Um novo projeto começa como **Aberto**. Se a categoria dele tiver um modelo de
agente de projeto, a Lotti também provisiona esse agente depois que o projeto é
salvo. O projeto continua funcionando normalmente quando não existe um modelo
correspondente.

Use um título que descreva o resultado ou a missão duradoura. “Estabelecer a
reserva lunar de sardinhas” diz mais do que “Sardinhas”.

## Editar configurações do projeto

A rota de configurações do projeto fornece o editor de formulário completo para
status, título, data prevista, saúde, atribuição de agente, alterações
propostas pelo agente e tarefas vinculadas. Ela também mantém as ações
**Cancelar** e **Salvar** fixadas na parte inferior, para que as alterações
pendentes do formulário fiquem explícitas.

## Entenda o status do projeto

| Status | Use quando |
| --- | --- |
| Aberto | O projeto existe, mas a execução ativa ainda não começou. |
| Ativo | O trabalho está em andamento e pertence ao planejamento do dia. |
| Monitoramento | Nenhum trabalho está agendado, mas o resultado ainda precisa de observação. |
| Em espera | O trabalho está pausado por um motivo conhecido. |
| Concluído | O resultado pretendido foi alcançado. |
| Arquivado | O projeto é mantido como histórico, mas não está mais operacional. |

Selecione a pílula de status na página de detalhes do projeto para alterá-lo.
Alterações de status, de categoria e de data prevista são salvas imediatamente;
não há um botão de salvar separado nessa superfície de detalhes voltada à
leitura.

## Leia a saúde do projeto e seu relatório

<ManualScreenshot
  caseId="projects/detail"
  alt="Detalhe do Project Waddle mostrando uma pontuação de saúde de 82, cinco das oito tarefas concluídas, um bloqueador e um relatório de IA atualizado há duas horas"
  caption="O desktop mantém a lista de projetos e os detalhes lado a lado. O celular abre os mesmos detalhes como uma página inteira."
/>

O painel de saúde resume a avaliação atual do agente do projeto:

- a pontuação de saúde é um sinal compacto, não um substituto para o trabalho
  em si;
- o progresso das tarefas mostra as contagens de concluídas e bloqueadas;
- o relatório traz um resumo atual e pode se expandir para o conteúdo completo;
  e
- as recomendações chamam a atenção para próximas ações concretas.

No Project Waddle, uma pontuação de 82 e cinco tarefas concluídas são
animadoras, mas o alimentador de peixes com gravidade zero ainda é um
bloqueador do lançamento. Leia a explicação antes de tratar uma pontuação como
veredicto.

Quando um agente de projeto está configurado, o menu do relatório pode
solicitar um relatório novo ou cancelar um despertar agendado. Sem agente, o
projeto ainda mostra suas tarefas e qualquer texto de projeto armazenado.

## Mantenha o trabalho do projeto nas tarefas

<ManualScreenshot
  caseId="projects/tasks"
  alt="Painel de tarefas do Project Waddle listando a inspeção do habitat, a calibração do alimentador de peixes em gravidade zero e a confirmação das cápsulas de carga de sardinhas, com estimativas e status"
/>

As linhas de tarefas do projeto mostram o título de cada tarefa, um resumo
curto, a estimativa e o status atual. Selecione uma linha para abrir a própria
tarefa. Os detalhes da tarefa continuam sendo a fonte da verdade para
descrições, gravações, listas de verificação, imagens, registros de tempo e
arte de capa.

Atribua uma tarefa a um projeto pelo controle de projeto no cabeçalho da
tarefa. Os projetos têm escopo por categoria, então dê uma categoria à tarefa
antes de escolher um projeto. Uma tarefa pertence a no máximo um projeto por
vez; atribuir outro projeto move o vínculo em vez de duplicá-lo.

## Projetos, categorias e tarefas

Use os três níveis deliberadamente:

- **Categoria:** uma área operacional duradoura, como Operações Penguin.
- **Projeto:** um resultado com várias tarefas, como o Project Waddle.
- **Tarefa:** uma unidade executável, como recalibrar o alimentador de peixes
  com gravidade zero.

Evite criar um projeto para uma única ação simples. Use um quando várias
tarefas precisarem de uma data prevista, um status, um consolidado de progresso
ou um relatório operacional compartilhados.
