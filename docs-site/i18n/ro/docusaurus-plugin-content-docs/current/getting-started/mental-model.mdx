## Elementele de bază, fără a pretinde că sunt toate la fel

| Element | Îl folosiți pentru | Exemplu |
| --- | --- | --- |
| Categorie | Un domeniu important al vieții sau muncii | Sănătate, familie, cercetare |
| Sarcină | Un rezultat care are nevoie de progres | Pregătiți lansarea V1 |
| Listă de verificare | Pași concreți într-o sarcină | Verificați notele de lansare |
| Înregistrare | Context sau dovezi | Notă, imagine, înregistrare audio |
| Bloc planificat | Un interval propus în zi | Revizuirea lansării, 10:00–11:00 |
| Înregistrare de timp | Ce s-a întâmplat în realitate | Revizuirea lansării, 10:18–11:07 |
| Obicei | Un comportament recurent pe care îl evaluați | Luați medicamentele |
| Măsurătoare | O observație numerică | Puls în repaus |
| Tablou de bord | O perspectivă asupra măsurătorilor utile | Privire de ansamblu asupra antrenamentului |

## Agenții sunt invitați, nu coproprietari

Un agent poate citi contextul pe care i-l oferiți, poate forma o opinie, poate rezuma o sarcină complicată și poate propune următoarea schimbare utilă. Raportul său este o opinie cu proveniență, nu un fapt nou în istoricul dvs. Modificările propuse pentru sarcini, liste de verificare, stări sau date așteaptă să le confirmați ori să le respingeți. Excepțiile restrânse sunt un titlu sau o limbă inițială pentru o sarcină altfel goală; după aceea se aplică aceeași regulă a aprobării umane.

Aceasta nu este doar politețe adăugată unui prompt. Lotti separă jurnalul personal de memoria de lucru a agentului, astfel încât agentul nu vă poate trata istoricul ca pe o ciornă. Puteți folosi AI, îl puteți ajusta sau îl puteți lăsa complet deoparte.

## O primă orbită practică

1. Creați categorii pe care chiar le folosiți când luați decizii.
2. Capturați munca sub formă de sarcini și păstrați materialele de sprijin atașate lor.
3. Folosiți Daily OS ca să decideți ce aparține unei anumite zile.
4. Înregistrați timpul real și observațiile fără să le forțați să se potrivească planului.
5. Consultați obiceiurile și tablourile de bord pentru tipare, nu pentru zile perfecte luate separat.

## Păstrați clară proprietatea

- O sarcină legată este proprietara titlului și categoriei sale.
- Un bloc Daily OS este proprietarul poziției și duratei sale.
- O înregistrare de timp este proprietara relatării a ceea ce s-a întâmplat.
- AI-ul propune; dvs. analizați și decideți.

:::tip[Scopul nu este o cronologie impecabilă]

Dacă planul s-a schimbat, lăsați-l să se schimbe. Dacă ziua a luat-o într-o direcție neașteptată, consemnați ce s-a întâmplat. Un sistem util vă ajută să vedeți forma vieții dvs.; nu vă cere să jucați pentru baza de date rolul unei persoane mai productive.

:::
