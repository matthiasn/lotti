# Gestionați munca prin proiecte

Proiectele se află între categorii și sarcini. O categorie precum **Operațiuni cu pinguini** definește un domeniu larg de responsabilitate; **Project Waddle** grupează sarcinile concrete necesare pregătirii habitatului orbital pentru pinguini.

## Găsiți proiectul necesar

Căutarea se potrivește cu titlurile proiectelor, textul stocat și numele categoriilor. Controlul de filtrare restrânge lista după starea proiectului sau categorie. Un filtru rămâne inactiv până când selectați **Aplicați**.

<ManualScreenshot
  caseId="projects/filters"
  alt="Dialogul de filtrare a proiectelor, cu controale pentru stare și categorie peste lista de proiecte Intergalactic Penguin Logistics"
/>

Proiectele rămân grupate pe categorii după filtrare. Aceasta face inițiativele cu nume similare mai ușor de distins și păstrează vizibil fiecare domeniu operațional.

## Creați un proiect

Selectați butonul **+** în Proiecte, apoi introduceți:

- un titlu obligatoriu;
- o categorie opțională; și
- o dată-țintă opțională.

<ManualScreenshot
  caseId="projects/create"
  alt="Formularul de creare a proiectului pentru Înființați rezerva lunară de sardine în categoria Operațiuni cu pinguini"
  caption="Formularul de creare este o foaie inferioară pe mobil și un dialog centrat pe desktop."
/>

Un proiect nou începe în starea **Deschis**. Dacă categoria lui are un șablon de agent pentru proiecte, Lotti asigură și acel agent după salvare. Proiectul funcționează în mod normal și fără un șablon potrivit.

Folosiți un titlu care descrie rezultatul sau misiunea durabilă. „Înființați rezerva lunară de sardine” spune mai mult decât „Sardine”.

## Editați setările proiectului

Ruta setărilor proiectului oferă editorul complet pentru stare, titlu, dată-țintă, sănătate, atribuirea agentului, modificările propuse de agent și sarcinile legate. Păstrează și acțiunile **Anulați** și **Salvați** fixate jos, astfel încât modificările formularului aflate în așteptare să fie explicite.

## Înțelegeți starea proiectului

| Stare | Folosiți-o când |
| --- | --- |
| Deschis | Proiectul există, dar execuția activă nu a început. |
| Activ | Munca avansează acum și aparține planificării zilei. |
| Monitorizare | Nu este programată muncă, dar rezultatul încă are nevoie de observație. |
| În așteptare | Munca este suspendată dintr-un motiv cunoscut. |
| Finalizat | Rezultatul intenționat a fost atins. |
| Arhivat | Proiectul este păstrat pentru istoric, dar nu mai este operațional. |

Selectați pastila stării din pagina de detalii a proiectului pentru a o schimba. Modificările de stare, categorie și dată-țintă se salvează imediat; pe această suprafață de detalii nu există un buton separat de salvare.

## Citiți sănătatea proiectului și raportul său

<ManualScreenshot
  caseId="projects/detail"

  caption="Desktopul păstrează lista de proiecte și detaliul alături. Mobilul deschide același detaliu pe o pagină întreagă."
/>

Panoul de sănătate rezumă evaluarea curentă a agentului de proiect:

- scorul de sănătate este un semnal compact, nu un substitut pentru munca de bază;
- progresul sarcinilor arată numărul de sarcini finalizate și blocate;
- raportul oferă un rezumat actual și poate fi extins la conținutul complet; iar
- recomandările atrag atenția asupra următoarelor acțiuni concrete.

În Project Waddle, un scor de 82 și cinci sarcini finalizate sunt încurajatoare, însă alimentatorul de pești în imponderabilitate rămâne un blocaj de lansare. Citiți explicația înainte de a trata un scor ca pe un verdict.

Când este configurat un agent de proiect, meniul raportului poate solicita un raport proaspăt sau poate anula o activare programată. Fără agent, proiectul își arată în continuare sarcinile și orice text de proiect stocat.

## Păstrați munca proiectului în sarcini

<ManualScreenshot
  caseId="projects/tasks"
  alt="Panoul de sarcini Project Waddle, cu inspecția habitatului, calibrarea alimentatorului de pești în imponderabilitate și confirmarea capsulei de marfă cu sardine, cu estimări și stări"
/>

Rândurile de sarcini ale proiectului arată titlul, rezumatul scurt, estimarea și starea curentă a fiecărei sarcini. Selectați un rând pentru a deschide sarcina. Detaliile sarcinii rămân sursa adevărului pentru descrieri, înregistrări, liste de verificare, imagini, înregistrări de timp și coperți.

Asociați o sarcină unui proiect din controlul proiectului din antetul sarcinii. Proiectele sunt delimitate de categorie, deci dați sarcinii o categorie înainte de a alege proiectul. O sarcină aparține cel mult unui proiect simultan; alegerea altui proiect mută legătura, nu o dublează.

## Proiecte, categorii și sarcini

Folosiți cele trei niveluri în mod deliberat:

- **Categorie:** un domeniu operațional durabil, precum Operațiuni cu pinguini.
- **Proiect:** un rezultat cu mai multe sarcini, precum Project Waddle.
- **Sarcină:** o unitate executabilă, precum recalibrarea alimentatorului de pești în imponderabilitate.

Evitați crearea unui proiect pentru o singură acțiune simplă. Folosiți unul când mai multe sarcini au nevoie de o dată-țintă comună, stare, progres cumulat sau raport operațional.
