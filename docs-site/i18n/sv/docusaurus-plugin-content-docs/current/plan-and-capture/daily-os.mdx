# Planera och avsluta en dag med Daily OS

Daily OS omvandlar en muntlig eller inskriven incheckning till en plan för en dag. Det gör det inte
Byt ut uppgifter eller tidsanteckningar: Uppgifter beskriver arbetet, dagplanen beskriver
avsikt och inspelade sessioner beskriver vad som faktiskt hände.

Exemplen följer direktör Aurora genom en medvetet hektisk dag på
Intergalaktisk pingvinlogistik. Planeraren skyddar en lansering av Project Waddle
Recension, flyttar sardinförhandlingar och håller en gravitationsfri fiskmatningsdemonstration
Klart. Din pingvinsituation kan variera.

## Innan din första incheckning

Öppna **Inställningar → Daily OS**. Välj den inferensprofil som ska planeras
dina dagar och ange namnet Daily OS bör använda när de tilltalar dig. The
Integritetsmeddelande identifierar den valda leverantören och om behandlingen är påslagen
Den här enheten eller fjärrkontrollen.

<ManualScreenshot
  alt="Daily OS settings configured for Director Aurora and the Project Waddle inference profile"
  caseId="daily-os/settings"
  caption="Director Aurora uses the Project Waddle Command profile, Waddle Command 70B, and the Mission Control Router provider."
/>

Öppna sedan **Inställningar → Kategorier** och aktivera **Dagplanering** endast för
Kategorier som planeraren kan använda. En avsiktlig tillåtningslista håller orelaterad
uppgifter utanför planeringen och förhindrar att Daily OS ändrar arbete utanför dessa
kategorier.

## 1. Säg vad dagen behöver

Öppna **Daily OS**, välj datum och välj **Speak a check-in**. Nämn:

-  fasta åtaganden och tidsfrister;
- de resultat som spelar roll;
-  ungefärliga varaktigheter och tillgänglig kapacitet;
- avbrott, energibegränsningar och arbete som inte borde ske för sent;
- något som redan pågår eller sannolikt kommer att föras över.

Välj **Skriv istället** när du talar är besvärligt. Efter erkännande,
Gå igenom hela utskriften och rätta namn, nummer eller andra misstag
på plats.

<ManualScreenshot
  alt="The complete Daily OS check-in transcript ready for review"
  caseId="daily-os/capture-review"
  caption="Review the recognized text before Daily OS matches it with Project Waddle tasks."
/>

Välj **Spela in igen** för att börja om eller **Recensera** för att fortsätta. Du kan inte bygga
dagen tills matchningspasset är klart.

## 2. Förena tal med verkligt arbete

Försoning är säkerhetskontrollen mellan vad assistenten hörde och vad Lotti har hört
kommer att förändras:

- ** Matched** pekar på en befintlig uppgift. Bekräfta att det är uppgiften du menade.
- **Ny** är ett uttryck som kan bli ett verk för idag.
- **Uppdatering** föreslår en ändring av en befintlig uppgift.
- Självförtroende och tidsankaretiketter visar var granskning är mest användbar.

<ManualScreenshot
  alt="Daily OS reconciling a spoken check-in with existing and new Project Waddle work"
  caseId="daily-os/reconcile"
  caption="The orbital habitat inspection is matched to real work while new requests stay explicit decisions."
/>

Daily OS visar också arbete som pågår, är försenat, ska lämnas in idag eller
Nyligen missad. Bestäm vad som hör hemma idag istället för att importera hela
Eftersläpning.

## 3. Bedöm värnplikten

Välj **Bygg min dag** efter att försoningsbesluten ser rätt ut. The
resultatet öppnar på **Agenda**. Kapacitetskortet svarar "passar det här?" innan
Den numrerade listan svarar "Vad spelar roll?" Uppgiftsbaserade rader håller sig verkliga
Kategori, status, uppskattning och omslagsminiatyr.

<ManualScreenshot
  alt="A Project Waddle Daily OS agenda with capacity, task thumbnails, priorities, and category totals"
  caseId="daily-os/agenda"
  caption="Every task in the agenda keeps its own Project Waddle cover art, including completed work."
/>

Använd de snabba justeringarna när formen är fel:

- ** För mycket** kräver en mindre plan.
- **Flytta lättare** flyttar krävande arbete bort från lågenergiperioder.
- **Lägg till buffert** skapar andrum.
- ** Förfina** öppnar en fri röst- eller textjustering.

Välj **Ser bra ut** bara när utkastet verkligen är acceptabelt.

## 4. Granska varje föreslagen förfining

Refine skriver aldrig tyst om schemat. Beskriv förändringen, och inspektera sedan
varje tilllade, flyttade eller borttagna block med sin gamla och föreslagna tid och
Planerarens anledning. Acceptera eller avvisa förslag individuellt, välj **Återställ** till
Släng den väntande setet, eller fortsätt prata för att be om ett nytt pass.

<ManualScreenshot
  alt="Daily OS proposing two reviewed time changes for sardine negotiations and the fish-feeder demo"
  caseId="daily-os/refine"
  caption="Sardine negotiations move later to protect Mission Control's launch review; the fish-feeder demo follows the new buffer."
/>

Endast accepterade förslag ingår i utkastet. **Ser bra ut** lämnar
Gå igenom när planen visar vad du tänker.

## 5. Lås in dagen

**Lås in det**-sidan ger hela utkastet en sista läsning: ordning,
varaktighet, resultat och återstående kapacitet. Håll bekräftelsekontrollen för
en sekund för att flytta planen från utkast till genomförd. Tangentbordsanvändare kan fokusera
styra och trycka Enter eller mellanslag; Aktivering av hjälpmedelsteknologi bekräftas
det direkt.

<ManualScreenshot
  alt="The Project Waddle draft and deliberate hold-to-confirm action on the Daily OS commitment page"
  caseId="daily-os/commit"
  caption="Eight pieces of work use 8h 45m of a 9h day, leaving a 15-minute margin before Director Aurora signs off."
/>

Att sätta lås i planens ben, inte din förmåga att anpassa dig. Du kan fortfarande
använd **Förfina** efteråt, men föreslagna ändringar kan fortfarande granskas.

## 6. Jämför planen med verkligheten

Byt från **Agenda** till **Dag** för kalenderprojektionen. På skrivbordet,
Planera och faktiskt dela en tidsaxel. På en telefon, svep horisontellt mellan dem
de planerade och inspelade filerna. Planerade block använder en lättare behandling och
Inspelade sessioner fylls upp, så att driften förblir synlig utan omskrivning
historia.

<ManualScreenshot
  alt="Daily OS timeline comparing Project Waddle planned blocks with recorded time"
  caseId="daily-os/timeline"
  caption="The phone pages between planned and recorded lanes; desktop shows both on one time axis."
/>

Starta och stoppa tidsregistreringen från den länkade uppgiften som vanligt. Dagliga OS-läsningar
dessa register som verklighet; den tillverkar inte färdigt arbete från
planera.

## 7. Flytta, ändra storlek eller redigera ett block

Välj den fyrpiliga **Arrangera**-åtgärden ovanför tidslinjen. Dra en blockkropp
för att flytta den, eller dra dess övre eller nedre handtag för att ändra dess början eller slut.
Ändringar snabbar upp till 15-minutersintervaller och håller sig inom den valda dagen. Användning
**ångra** i bekräftelsemeddelandet när den gamla platsen var bättre.

<ManualScreenshot
  alt="Daily OS arrange mode with move and resize controls on Project Waddle blocks"
  caseId="daily-os/arrange"
  caption="Arrange mode exposes the same direct move and resize actions on phone and desktop."
/>

Pennmekanismen öppnar blockredigeraren. Ett uppgiftslänkat block behåller sitt
uppgiftsägd titel och kategori endast läsbar; välj **Öppna uppgift** för att ändra dessa
vid källan. Dess planerade start och slut kan fortfarande redigeras från Daily OS.

<ManualScreenshot
  alt="Daily OS block editor for the zero-gravity fish feeder task"
  caseId="daily-os/block-editor"
  caption="The linked task owns its title and Penguin Operations category; Daily OS owns the 2:30–4:00 PM slot and planning reason."
/>

Fristående planblock kan redigera sin egen titel och kategori eftersom det inte finns någon uppgift
äger den metadatan.

## 8. Avsluta dagen

Efter en engagerad dag, välj **Avsluta av**. Nedstängning skiljer beslut som
bör inte förväxlas:

- **Vad du gjorde** listar färdigt arbete med faktisk spårad längd.
- **Carries forward** låter dig schemalägga ofärdigt arbete till imorgon, välj en
  Dejta, eller släpp det.
- Mätvärden sammanfattar fokustid, flödessessioner, kontextbyten och energi.
- En enradig reflektion matar nästa utkast.
- **Det jag har lärt mig** låter dig bekräfta eller glömma kunskap om varaktiga planerare.
- **Till imorgon** bevarar den kortfattade överlämningsanmärkningen.

<ManualScreenshot
  alt="Daily OS shutdown with completed Project Waddle work, carryover choices, metrics, reflection, and learned preferences"
  caseId="daily-os/shutdown"
  caption="Director Aurora completed the habitat inspection, launch review, and sardine deal; the fish-feeder demo and passenger question stay explicit carryover decisions."
/>

Välj **Spara & stäng** för att lämna recensionen utan att deklarera dagen stängd,
eller **Stäng dagen** när besluten om att ta över och lära sig är klara.

## Ägarregeln

- Uppgifter beskriver arbetet och metadata för den egen uppgiften.
- Dagsplanen beskriver avsikt och äger schemalagda block.
- Tidsregister beskriver verkligheten.
- Planerarens kunskapsregister granskade preferenser för framtida tider.
- Assistenten friar; Du bestämmer.
