# Underhåll denna manual

Källkoden finns i applikationsarkivet, så en beteendeförändring och dess
Dokumentation kan landa ihop. Genererade skärmdumpar finns i syskonet
`lotti-docs` repository and are never committed to the application tree.

## Redigera prosa

1. Edit or add the English MDX file under `docs-site/docs`.
2. Uppdatera matchningsfilen för varje publicerad plats under
   `docs-site/i18n/<locale>/docusaurus-plugin-content-docs/current`.
3. Add a new page to `docs-site/sidebars.ts`.
4. Update `docs-site/metadata/features.json` when coverage changes.
5. Uppdatera matchande poster i
   `docs-site/metadata/surface-inventory.json`. A surface becomes **verified**
   endast när dess nuvarande produktionsgränssnitt, prosa och skärmdumpstäckning har
   Alla har granskats.
6. Run `make manual_check`.
7. Review the production build locally with `make manual_serve`, including the
   Språkväljare och varje lokaliserad rutt.

Validering kräver att varje publicerat översättningsträd innehåller samma sida
stigar som det engelska källträdet. Den avvisar också översatta sidtexter som
är identiska med engelska, så nya sidor kan inte tyst skickas som oöversatta
kopior. De offentliga engelska URL:erna förblir oförändrade; Andra språk är
publicerade under sitt eget versionerade platsprefix.

## Håll täckningsinventariet auktoritativt

Inventariet definierar V1:s engelska-manualgräns: varje Beamer-sida, varje
Inställningar V2 blad eller editor, och alla större arbetsflöden som skapar eller material
ändrar data eller tvärskärande konfiguration. Varje post registrerar sitt ägande
källfil och ett källtextankare. Validering misslyckas om någon av dem försvinner,
vilket tvingar fram en rutt- eller UI-ändring för att uppdatera den manuella revisionen istället för att lämna
En osynlig, gammal bråk.

Täckningsstater har medvetet strikta betydelser:

- **Planned** betyder att ytan är känd men fortfarande saknar fullständig täckning.
- **Dokumenterat** betyder att användbar prosa finns, men skärmdumpar från nuvarande produktion
  eller så saknas en noggrannhet från början till slut.
- ** verifierad betyder att den relevanta funktionssidan är verifierad och stödd av
  Registrerade produktionsskärmdumpar.

Run `npm --prefix docs-site run coverage:complete` for the release gate. It
misslyckas tills varje inventerad yta är verifierad; Vanlig inkrementell manual
Builds fortsätter att rapportera de återstående räkningarna utan att blockera ämnesstorlek
Åtaganden.

## Lägg till ett skärmdumpsfall

1. Lägg till eller återanvänd en deterministisk Flutter-skärmdumpssele.
2. Registrera fallet och dess fyra källbilder i
   `docs-site/metadata/screenshot-cases.json`.
3. Put visible fixture copy behind `manualScreenshotText(...)` for every
   Stödd lokalitet när den inte redan tillhandahålls av applokaliseringen
   filer.
4. Capture every locale into `../lotti-docs`; never into this repository.
5. Skapa och validera mediemanifestet.
6. Reference the case with `<ManualScreenshot caseId="…" alt="…" />` on
   every language page.

Every app screenshot in an authored page must use `ManualScreenshot`. Direct
links to old `lotti-docs` images fail manual validation because they cannot
follow the global Mobile/Desktop choice or the manual's light/dark theme.

Every automated case must provide mobile-light, mobile-dark, desktop-light,
and desktop-dark variants in every published locale. The manual component
chooses the matching language and theme. Changing Mobile/Desktop on any image
updates every screenshot immediately and persists across pages and open manual
tabs.

## Publish a release

Manual source is not copied for every app version. The app release tag already
preserves its exact source. CI checks out that tag, builds with
`MANUAL_VERSION=<version>`, and publishes an immutable static directory. The
release dropdown is a small manifest of those directories.
