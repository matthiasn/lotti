## Generative (Glados) testing opportunities

- [x] **[HIGH]** `VectorClock.merge(vc1, vc2)` — has clear algebraic invariants:
  - Commutativity: `merge(a, b) == merge(b, a)` (uses `max` per node; order-independent).
  - Idempotency: `merge(a, a) == a`.
  - Result dominance: for each node `k`, `result.get(k) >= a.get(k)` and `result.get(k) >= b.get(k)`.
  All are genuine Glados properties over the existing `any.vc` / `any.vc3` generators. Currently `merge` has **zero test coverage**.
- [x] **[HIGH]** `VectorClock.mergeUniqueClocks(Iterable<VectorClock?>)` — properties:
  - Returns `null` iff all inputs are `null`.
  - Result contains no duplicates (`result.toSet().length == result.length`).
  - Every non-null input element appears in the result.
  Good Glados candidate using a `List<VectorClock?>` generator.
- [ ] **[MED]** `VectorClock.fromJson` / `toJson` round-trip: `fromJson(vc.toJson()).vclock == vc.vclock`. Straightforward property over `any.vc`.
