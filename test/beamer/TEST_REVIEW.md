  requiring a split).
- **2 oversized test files**: `test/beamer/beamer_app_test.dart` (2155 lines — most urgent, split

  desktop group).

  (5 tests with zero production coverage), `isA<Widget>()` / "type check omitted" in
  `settings_location_test.dart`, missing `FillSurveyWithTypePage` type assertion in
  `journal_location_test.dart`.
- **2 genuine Glados candidates**: `_dailyOsNextRouteFrom(Uri)` (parse/reject invariants) and
  `calculateClampedIndex(...)` (clamp algebraic properties). Neither has any property-based tests
  today.
- **4 speed/reliability wins**: 10 `pumpAndSettle` calls in `not_logged_in_toast_test.dart`
  (highest risk), 3 s virtual-time pumps in `my_beamer_app_test.dart`, repeated full GetIt
  container setups in `beamer_app_test.dart`, and 7 copy-pasted viewport setup blocks.
- **Biggest single opportunity**: Split `test/beamer/beamer_app_test.dart` (2155 lines) into
  three files sharing a helper module — this alone halves the largest test file in the scope and
  makes the groups individually navigable.
