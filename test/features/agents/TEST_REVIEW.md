## Recurring patterns

- **Monolithic test files that dwarf their impl**: The single most-repeated problem across the feature. Files like `agent_repository_test.dart` (7 412 lines), `task_agent_workflow_test.dart` (6 057 lines), `wake_orchestrator_test.dart` (5 659 lines), and `template_evolution_workflow_test.dart` (5 527 lines) each exceed the preferred ceiling by 5â€“12Ă—. In every case, natural `group()` seams already exist that map directly to proposed split files.

- **Mock setup dwarfing test logic**: Repeated verbatim stub blocks (39 copies of a 3-line session-stub in `evolution_chat_state_test.dart`; 38 copies of `getAgentsForTemplate` in `agent_template_detail_page_test.dart`; 12+ copies of `_stubCreateVersion` in `template_evolution_workflow_test.dart`) inflate file size and obscure the actual assertion logic.

- **Mock objects rebuilt on every Glados iteration**: Across service/, tools/, state/, sync/, and wake/, `setUp` rebuilds mock objects and re-wires `when(...)` stubs for every Glados run. The pattern costs hundreds of redundant allocations per property test. Moving construction to `setUpAll` + `reset(mock)` is the standard fix and is the highest-ROI speed improvement in the feature.

- **`pumpAndSettle` overuse in UI tests**: Clusters of 33â€“74 `pumpAndSettle` calls in individual test files (`agent_soul_detail_page_test.dart` 74, `agent_template_detail_page_test.dart` and `agent_settings_page_test.dart` 57 each, `proposals_test.dart` ~46, `sidebar_wake_queue_test.dart` 33) introduce multi-second CI latency. Most intermediate navigation pumps can be replaced with bounded `tester.pump()`.

- **Fake-time policy violations**: `Future.delayed(Duration.zero)` (or equivalent microtask busy-wait loops) appear in `genui_event_handler_test.dart` (15 occurrences), `agent_providers_test.dart` (3), `soul_evolution_chat_state_test.dart` (1 + 4 microtask spin loops), and `evolution_chat_page_test.dart`. All should be replaced with `pumpEventQueue()` or deterministic `await Future<void>.value()`.

- **One-test-file-per-source-file violations**: `soul_document_serialization_test.dart` tests `agent_domain_entity.dart`; `soul_avatar.dart` is tested inside `agent_list_filter_state_test.dart`; `agent_repository` is spread across four satellite test files. Each case needs either formal adoption of the split strategy or consolidation.

- **Inline mock definitions and fallback values**: Isolated pockets (`evolution_recorder_test_utils.dart` inline `MockRealtimeService`, four tools test files with inline `registerFallbackValue`, `agent_sync_service_test.dart` inline GetIt manipulation, `project_agent_workflow_test.dart` inner `_MockConversationRepository`) violate the centralized-mocks and centralized-fallbacks rules and create divergence risk when new fallback types are introduced.

- **Missing tests for pure-logic pure functions**: `truncateAgentText`, `_classifyTextSentiment`, `_argsContainExplanatoryContext`, `_parsePriority`, `_deduplicateChangeSets`, `aggregateByModel`, `_isImplicitApprovalMessage`, and `_parseProjectStatus`/`_isSameSemanticStatus` are all pure functions with structured inputs and algebraic invariants that have zero or only concrete example coverage. These are the highest-value Glados gaps in the feature.

- **`setSurfaceSize` repeated per-test in UI page tests**: `agent_templates_page_test.dart` (13 tests), `agent_instances_page_test.dart`, `agent_pending_wakes_page_test.dart`, `agent_souls_page_test.dart`, `agent_listing_shell_test.dart` all call `setSurfaceSize` per test when `setUpAll`/`tearDownAll` would suffice.
