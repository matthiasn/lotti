## Generative (Glados) testing opportunities

None of the genui test files use Glados. The subdirectory is UI/widget-heavy, so most of the code is a poor fit. However, there are a handful of pure-logic functions worth examining:

- [ ] **[MED]** `lib/…/genui/evolution_catalog_helpers.dart` — `readInt`, `readDouble`, `readString`, `readStringOrNull`, `readNumOrNull`, `readMapList` are pure functions whose behavior over arbitrary JSON maps (`Map<String, Object?>`) can be described as algebraic invariants:
  - `readInt(m, k)` returns a `num`-to-int coercion when the value is numeric, otherwise the fallback.
  - `readMapList` filters non-`Map` list items.
  - These are non-trivial because the input type is `Object?`, not a typed model, making exhaustive hand-rolled tests impractical. Glados over random `(String key, Object? value)` pairs would catch type-confusion edge cases (e.g., `bool`, nested `List`, `null` in a typed position).

- [ ] **[LOW]** `lib/…/genui/evolution_catalog.dart` — `_categoryBar` computes `neutralCount = max(0, count - positiveCount - negativeCount)` and `fraction = totalCount > 0 ? count / totalCount : 0.0`. These are pure arithmetic invariants (`fraction ∈ [0, 1]`, `neutralCount >= 0`) testable with Glados over random non-negative ints. The risk of regressions is low but the logic is non-trivial when `positiveCount + negativeCount > count`.

No Glados candidates exist for the widget builders themselves (UI, real I/O, localization context required).

## Test execution speed opportunities

- [ ] **[HIGH]** `test/…/genui/genui_event_handler_test.dart` — 15 occurrences of `await Future<void>.delayed(Duration.zero)` (lines 60, 78, 173, 194, 211, 228, 251, 268, 287, 311, 335, 360, 378, 397, 414, 434). This is a real-time async yield that violates the fake-time policy and introduces unnecessary scheduler-dependent ordering in CI. Replace each with `await pumpEventQueue()` (from `flutter_test`), which flushes the microtask queue deterministically. **Estimated impact: ~15 ms of avoidable real-time sleep per run, plus potential flakiness on slow CI.**

- [ ] **[MED]** `test/…/genui/ab_comparison_card_test.dart` — every test uses `pumpAndSettle()` (lines 35, 42, 50, 58, 66, 79, 82, 94, 97, 104, 107, 121, 124, 134, 137, 148, 152, 160) even for simple builds with no animations. `ABComparisonCard` does not use animated transitions; `tester.pump()` is sufficient for all cases except the `resets state` tests which rebuild and tap. Switching ~14 of the 18 `pumpAndSettle` calls to `pump()` removes the 10-second hang risk and speeds up the test.

- [ ] **[MED]** `test/…/genui/evolution_catalog_interactive_widgets_test.dart` — 18 `pumpAndSettle()` calls (lines 508, 511, 521, 542, 545, 554, 574, 577, 586, 605, 608, 617, 636, 639, 648, 666, 669, 676). The `ABComparisonCard` uses no `AnimatedCrossFade` or other indefinite animations; `DesignSystemButton` press is synchronous. Most of these can be replaced with `tester.pump()`. The only legitimate `pumpAndSettle` use is after tapping a `GestureDetector` that triggers `setState` + a `DesignSystemButton` state change, which is a single frame.

- [ ] **[LOW]** `test/…/genui/evolution_catalog_test.dart` — `pumpAndSettle` appears 7 times (lines 403, 421, 426, 931, 1030, 1114, 1756, 1759). The calls on lines 403, 421, 426 are inside the `EvolutionNoteConfirmation` expand/collapse tests which use `AnimatedCrossFade` — these are correct. The remaining calls (931, 1030, 1114) occur after button taps in the `CategoryRatings` and `BinaryChoicePrompt` tests. Since `setState` from a button press only requires one frame, `tester.pump()` is sufficient.

## Summary

- **1 oversized impl file** (`evolution_catalog.dart`, 1442 lines) and **1 oversized test file** (`evolution_catalog_test.dart`, 1770 lines) — both are prime split candidates.
- **4 weak test patterns** identified: 10 inline `CatalogItemContext` duplications, 5 copy-paste `didUpdateWidget` permutations, 2 copy-paste dispatch tests, and under-tested pure helpers in `evolution_catalog_helpers.dart`.
- **0 genuine Glados candidates** in the widget layer; **2 MED/LOW** candidates in pure parsing helpers (`readInt`/`readMapList` family and `_categoryBar` arithmetic).
- **3 coverage gaps**: the `rootType` fallback path in `genui_bridge.dart`, the `binary_choice_submitted` non-string-value branch in `genui_event_handler.dart`, and mixed-type filtering in `readMapList`.
- **2 speed wins**: replace 15 `Future<void>.delayed(Duration.zero)` calls in `genui_event_handler_test.dart` with `pumpEventQueue()`, and replace the majority of the ~36 `pumpAndSettle` calls across `ab_comparison_card_test.dart` and `evolution_catalog_interactive_widgets_test.dart` with `tester.pump()`.
- **Biggest opportunity**: split `evolution_catalog.dart` and `evolution_catalog_test.dart` along the schema / items / private-helpers seam — this single refactor would bring every file in the subdir below 500 lines and make it possible to navigate and review the test file without scrolling 1700+ lines.
