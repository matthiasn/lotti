## Generative (Glados) testing opportunities

The model subdir already has strong Glados coverage in 7 of 13 test files. The remaining genuine gaps are:

- [x] **[MED]** `lib/…/model/classified_feedback.dart` — `ClassifiedFeedbackItem` and `ClassifiedFeedback` serialization round-trips (`toJson` → `fromJson` → equality). - [x] **[MED]** `lib/…/model/proposal_ledger.dart` — `LedgerEntry.isOpen` depends only on `status == ChangeItemStatus.pending`. A Glados test over `ChangeItemStatus.values` would prove `isOpen` is `true` if and only if `status == pending`, closing the gap with 5 lines. **RESOLVED:** Stale — exhaustive per-status examples plus a Glados iff-property over `ChangeItemStatus.values` already exist in `proposal_ledger_test.dart`.

- [ ] **[LOW]** `lib/…/model/agent_domain_entity.dart` — `AgentSlots` serialization roundtrip. The struct has many optional `DateTime?` and `GCounter` fields. A Glados generator producing random combinations of set/unset optional fields would catch deserialization regressions in new slot fields more reliably than the current fixed-fixture tests.

- [ ] **[LOW]** `lib/…/model/seeded_directives.dart` — `SeedDirectiveChange.dateTime` parses the `date` string via `DateTime.parse`. - [ ] **[LOW]** `test/…/model/wake_run_time_series_utils_test.dart` — `numRuns: 180` (line 529). Same reasoning as above; 140 would be sufficient.

## Notes on test_data factories (`test/features/agents/test_data/`)

The 10 factory files total ~970 lines and are well-structured overall. Key observations:

- **DRY / quality:** All factories use `constants.dart` (`kTestAgentId`, `kTestTemplateId`, `kAgentTestDate`) consistently — good. The `entity_factories.dart` and `change_set_factories.dart` files expose defaults for every field, making factories maximally flexible without requiring callers to specify unrelated data.

- [x] **[MED] Missing factory:** `proposal_ledger.dart` defines `LedgerEntry` and `ProposalLedger`, but no factory exists in `test_data/` for them. Tests that need a `ProposalLedger` currently construct the object inline, which will lead to duplication as coverage grows. Adding `makeLedgerEntry(…)` and `makeProposalLedger(…)` to a new `ledger_factories.dart` would be consistent with the existing pattern. **RESOLVED:** Done — created `test_data/ledger_factories.dart` (full-parameter `makeLedgerEntry` + `makeProposalLedger`), exported from the `test_utils.dart` barrel, and `proposal_ledger_test.dart` now uses it instead of its local helper.

- [x] **[MED] `evolution_factories.dart`** imports from `wake_factories.dart` and `template_factories.dart` — cross-factory dependencies are present. This is fine but means factory changes can have transitive test breakage. Worth noting in the file-level docstring. **RESOLVED:** Done — file-level docstring added documenting the deliberate dependency on template/wake factories and the keep-defaults-stable guidance.

- [ ] **[LOW] `soul_factories.dart`** duplicates the `kTestSoulId` constant inline rather than exporting it from `constants.dart`. If other factory files or tests reference soul entities, they will need to redeclare or import from `soul_factories.dart` directly.

- [ ] **[LOW] Missing `observation_record` factory:** `ObservationRecord` appears in agent journal contexts but is not covered by any test-data factory. Tests that build observation-related entities (e.g., in `classified_feedback_test.dart`) construct the object inline.

## Summary

- **1 oversized impl file** (`seeded_directives.dart`, 657 lines — primarily const string data) that should be split.
- **0 oversized test files** (largest are 754 lines; below the 1000-line limit, though monitoring is warranted).
- **4 weak test quality issues**: `soul_document_serialization_test.dart` violates one-file-per-source rule; `agent_constants_test.dart` duplicates Glados enum coverage; `classified_feedback_test.dart` has only 2 tests with no behavioral coverage; `agent_token_usage_test.dart` has a toString smoke assertion with no value.
- **3 Glados candidates**: `classified_feedback.dart` enum round-trips, `proposal_ledger.dart` `isOpen` predicate, `AgentSlots` serialization with optional field combinations.
- **3 speed wins**: reduce `numRuns` in `agent_enums_test.dart` from 260 → 160, and trim `agent_time_utils_test.dart` / `wake_run_time_series_utils_test.dart` from 180 → 120–140.
- **Biggest opportunity**: the 8 impl files that have no test file at all (`pending_wake_record.dart`, `proposal_ledger.dart`, `observation_record.dart`, `seeded_directives.dart`, `hourly_wake_activity.dart`, `improver_slot_keys.dart`, `project_agent_report_contract.dart`, `ritual_summary.dart`) — several contain non-trivial pure logic (`PendingWakeRecord.id`, `LedgerEntry.isOpen`, `SeedDirectiveChange.dateTime`) that should be pinned with at least smoke-level tests.
