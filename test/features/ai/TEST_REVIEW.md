## Recurring patterns

- **Inline mock class definitions** appear in ~15 test files instead of using `test/mocks/mocks.dart`. The most duplicated are `MockUnifiedAiInferenceRepository` (×2), `MockTaskProgressRepository` (×3), `MockAiConfigDb` (×2), `FakeRequest` (×2), and multiple SSE/HTTP fakes.

- **Raw `getIt.isRegistered / unregister / registerSingleton` boilerplate** in `setUp`/`tearDown` appears in at least 15 test files. The project mandates `setUpTestGetIt()` / `tearDownTestGetIt()` for this purpose; the inline pattern is both verbose and a contamination risk under `very_good test`.

- **One-test-file-per-source rule violations**: at least 10 split test files exist against 5 implementation files (`ai_input_repository`, `ai_config_repository`, `prompt_builder_helper`, `inference_model_form_controller`, `inference_provider_form_controller`, `generated_prompt_card`, `ai_running_animation`). Half should be consolidated; the other half indicate that the implementation file itself should be split first.

- **`pumpAndSettle` overuse in widget tests**: counted across `ui/settings/` alone — over 200 `pumpAndSettle` calls where `tester.pump()` or `tester.pump(kThemeAnimationDuration)` would suffice. Each carries a 10-second default timeout, creating flakiness risk on slow CI runners.

- **`tester.runAsync` / real-time delays in tests**: `unified_ai_progress_view_test.dart` (7 `runAsync` blocks), `unified_ai_controller_test.dart` (mock answers with `Future.delayed(10ms)` inside `fakeAsync`), `whisper_inference_repository_test.dart` (real `sleep` for unique IDs). All are policy violations that add wall-clock time to CI.

- **Repeated mock stub boilerplate dwarfing test logic**: the 8-argument `when(() => mockOllamaRepo.generateTextWithMessages(...))` stub appears in 18+ locations in `conversation_repository_test.dart` alone; similar patterns in `ai_error_utils_test.dart`, `model_prepopulation_service_test.dart`, and `cloud_inference_repository_test.dart`. The principle "mock setup must not dwarf test logic" is violated in all four files.

- **Glados `numRuns` above guideline ceiling**: `task_due_date_handler_test.dart` (220 runs), `pcm_amplitude_test.dart` (200 runs ×2), `active_inference_controller_test.dart` (180 runs with list length 0–55). Reducing to 120–150 where the property space is well-explored saves ~10–30% Glados CI time per affected file.

- **Ad-hoc `MaterialApp + ProviderScope` wrappers** replacing `makeTestableWidget()`: appears in `unified_ai_progress_view_test.dart`, `unified_ai_popup_menu_test.dart`, `ai_error_display_test.dart`, `ai_core_functionality_test.dart`, `ai_settings_search_bar_test.dart`, `config_empty_state_test.dart`, `config_error_state_test.dart`, `form_components/ai_form_section_test.dart`, `form_components/ai_text_field_test.dart`, and others (~12 files). Each bypasses the standardised localization + theme setup.

- **Design-system token coupling in tests**: tests asserting exact pixel dimensions (`height: 48.0`), font weights (`w700`), letter spacings (`0.5`), and specific `EdgeInsets` values against design-system tokens. Any token update breaks these tests without any behavioral regression. Concentrated in `ai_settings_tab_bar_test.dart` (~20 assertions) and scattered through `config_empty_state_test.dart`/`config_error_state_test.dart`.
