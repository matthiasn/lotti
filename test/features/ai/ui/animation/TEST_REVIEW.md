## Test quality improvements

- [x] **[HIGH] `ai_animation_modal_interaction_test.dart` — inline mock `MockAiConfigDb extends Mock implements AiConfigDb`** (line 21): if a `MockAiConfigDb` exists in `test/mocks/mocks.dart` it must be removed from here; if it does not yet exist, add it there. **RESOLVED (already covered):** `MockAiConfigDb` already lives in mocks.dart and the test imports it from there.

- [x] **[HIGH] `ai_animation_modal_interaction_test.dart` — manual GetIt `registerSingleton` / `getIt.reset`** (lines 39–43, 63): violates the `setUpTestGetIt()` / `tearDownTestGetIt()` rule. The registrations of `DomainLogger`, `AiConfigRepository`, and `AiConfigDb` should go through the standard test helpers. **RESOLVED:** converted to `setUpTestGetIt(additionalSetup:)` / `tearDownTestGetIt`.

- [x] **[MED] `ai_running_animation_test.dart` — manual `container.dispose()` instead of `addTearDown`** (lines 105, 133, 184, etc.): the containers are disposed in the test body after the assertions, but if an assertion throws the dispose is skipped. Use `addTearDown(container.dispose)` immediately after creation (the pattern used in some but not all test groups in this file — see e.g. the `showExisting` tests in `unified_ai_progress_view_test.dart` for the correct pattern).
  **RESOLVED:** done (adapted) — a `makeContainer()` helper now registers `addTearDown(container.dispose)` as a safety net for every container, so a throwing assertion can no longer leak one. The in-body `container.dispose()` calls stay deliberately: they run *before* the binding's pending-timer check and cancel the cacheFor keep-alive timers, which addTearDown (running after the check) cannot do — pure addTearDown disposal was tried and fails with 'A Timer is still pending'. Riverpod's dispose() is a no-op on a disposed container, so the double dispose is safe (documented on the helper).

- [x] **[MED] `ai_animation_modal_interaction_test.dart` lines 92–152 — test "tapping animation opens progress modal when inference is active"**: the assertion only checks that `gesture.onTap isNotNull` — it does not actually tap and verify what happens. The test body should be evolved to verify the tap opens a modal (or fire and verify a specific provider action).
  **RESOLVED:** stale — during the merge the test was already evolved: it now taps the animation and asserts the progress modal actually opened (`find.text(testPrompt.name)` on the Wolt page title) with bounded pumps.

- [x] **[MED] `ai_running_animation_test.dart` lines 188–236 — test "should handle tap when interactive with active inference"**: taps the `GestureDetector` and then asserts only `find.byType(AiRunningAnimation)` — which proves nothing changed. ## Generative (Glados) testing opportunities

- [x] **[HIGH] `AiVoiceInputShaderPainter.shouldRepaint` and `AiThinkingLineShaderPainter.shouldRepaint`** — both painters implement `shouldRepaint` as a field-equality check (return `oldDelegate.X != X || ...`). This is pure boolean logic over ~8-10 named parameters. A Glados property test could verify the invariant: for any two painter instances where all fields are equal, `shouldRepaint` must return `false`; if exactly one field differs, it must return `true`. The current tests cover only 3–4 hand-picked change points. Generator shape: `_GeneratedPainterFields` combining all numeric/color/enum parameters via `combine9`. **RESOLVED:** Glados properties added for both painters — equal fields never repaint; bumping any single field (seed-driven, per-field perturbation) always repaints (120 runs each).

- [x] **[MED] `AiStateShaderProgramCache` load-and-reuse identity** — the cache memoises by returning the same `Future` instance. A property: calling `loadVoiceInput()` N times must always return `identical` objects. The current test only covers N=2. Glados with `any.positiveIntOrZero.map((n) => n % 8 + 2)` (2–9 calls) would cover more. This is simple but pure logic.
  **RESOLVED:** done (adapted) — exhaustive loop over N = 2..9 calls per fresh cache instead of Glados (small finite space): every call of `loadVoiceInput()`/`loadThinkingLine()` must return the `identical` memoised object.

- [ ] **[LOW] `AiVoiceShaderRoute` and `AiThinkingShaderRoute` label extensions** — both expose a `label` getter that returns a non-null, non-empty string for each enum variant. A trivial property: `for all route in AiVoiceShaderRoute.values, route.label.isNotEmpty` — but this is so simple it is better covered by a standard unit test loop than by Glados.
