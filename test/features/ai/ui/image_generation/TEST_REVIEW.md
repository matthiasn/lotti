## Test quality improvements

- [ ] **[HIGH] `reference_image_selection_widget_test.dart` — inline mock/fake classes** (lines 15–69): three controller classes `_MockReferenceImageSelectionController`, `_TrackingMockReferenceImageSelectionController`, and an unnamed processing variant (`_MockReferenceImageSelectionControllerWithProcessing` in `cover_art_skill_modal_test.dart`) are defined inline in the test files. Per AGENTS.md, if these mocks exist elsewhere in the test tree they must be centralised. At minimum, the shared shape should live in a `test/features/ai/image_generation/test_utils.dart` file and be imported by both test files.

- [ ] **[HIGH] `reference_image_selection_widget_test.dart` — manual GetIt setup/teardown** (lines 97–116): uses `getIt.reset()`, `getIt.allowReassignment = true`, and `getIt.registerSingleton<Directory>` inline rather than `setUpTestGetIt()`. AGENTS.md forbids inline `getIt.isRegistered / unregister / registerSingleton` boilerplate.

- [ ] **[HIGH] `cover_art_skill_modal_test.dart` — `setUpTestGetIt()` is used** (line 29) but immediately followed by a manual `getIt.registerSingleton<Directory>` (line 33): the additional registration is fine, but the directory singleton should be managed via `addTearDown` rather than relying solely on `tearDownTestGetIt` to clean up custom singletons that `tearDownTestGetIt` may not handle.

- [ ] **[MED] `cover_art_skill_modal_test.dart` — test "passes categoryId through to widget"** (lines 300–330): the assertion `expect(modal.categoryId, 'cat-123')` reads a widget property directly instead of asserting on a visible rendered effect. This is a weak assertion — it proves the parameter was forwarded in memory but not that it changes any rendered output or downstream behaviour. Prefer testing the rendered outcome (e.g., the category-specific image-fetching path being triggered).

- [ ] **[MED] `reference_image_selection_widget_test.dart` — `setUp()` creates a real temp directory** per test: while real file I/O in widget tests is acceptable for tests validating I/O behaviour, the majority of tests in this file do not need real image files. Only image-rendering tests (if any) need a real directory; tests that only check loading/error/callback state should use an empty stub directory.

- [ ] **[LOW] `reference_image_selection_widget_test.dart` — `pumpAndSettle()` in the "calls onSkip when no images are available" test** (line 164): `onSkip` is invoked from a `WidgetsBinding.addPostFrameCallback`. One `tester.pump()` suffices to trigger the post-frame callback; `pumpAndSettle()` is unnecessary and will block the full animation-settle timeout if any child animation is introduced.

- [ ] **[LOW] `cover_art_skill_modal_test.dart` line 70** — `expect(find.byType(CircularProgressIndicator), findsOneWidget)` as the sole assertion in "renders ReferenceImageSelectionWidget with correct taskId": this confirms the loading-state indicator renders but does not verify the `taskId` is actually forwarded. The test name does not match what is asserted. Either rename the test or add an assertion that the controller was built with the correct `taskId`.

## Test execution speed opportunities

- [ ] **[MED] `reference_image_selection_widget_test.dart` — real temp-directory creation/deletion in `setUp`/`tearDown` every test** (~17 tests): each test creates + deletes a temp directory even when no file I/O occurs. Extract a shared `setUpAll` temp directory that is only deleted once in `tearDownAll`.

- [ ] **[MED] `cover_art_skill_modal_test.dart` — three sequential `pump()` calls** in several tests (lines 113–118, 198–200): this pattern is used to drive the auto-skip post-frame callback. Document explicitly as `// pump 1: build; pump 2: post-frame callback; pump 3: state rebuild` for clarity, and confirm no test needs more than 2 pumps (the 3rd pump may be a defensive over-pump adding latency).

- [ ] **[LOW] `cover_art_skill_modal_test.dart` line 29 — `setUpTestGetIt()` is `await`-ed**: this is correct; no performance concern. The subsequent `tearDownTestGetIt()` should also be awaited — verify this is not a bare `tearDownTestGetIt()` without `await` (common mistake that hides cleanup race conditions).
