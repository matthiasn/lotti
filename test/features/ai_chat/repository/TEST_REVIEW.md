## Generative (Glados) testing opportunities

- [x] **[MED]** `ChatMessageProcessor.buildPromptFromMessages` and `buildFinalPromptFromMessages` are pure deterministic string-assembly functions. A Glados property could assert that for any sequence of user/assistant/tool messages the output always contains each message's content exactly once and in the correct role prefix order, and that concatenation of chunks never produces partial tokens. This would replace the current 5 hand-picked fixture tests and catch edge cases like empty strings, newlines in content, or Unicode.
  **RESOLVED:** done — added two Glados properties (numRuns 150 each) over seed-driven role sequences (user/assistant/tool): every message renders exactly once with its role prefix in input order, `buildPromptFromMessages` ends with the new 'User:' line, and `buildFinalPromptFromMessages` ends with the fixed closing instruction. The 5 fixture tests stay as readable worked examples.

- [x] **[MED]** `ChatMessageProcessor.accumulateToolCalls` is pure accumulation logic (no I/O). A Glados property over sequences of `(index, id?, arguments-chunk)` could assert: for any sequence, the final tool call list has exactly one entry per unique effective id, and the accumulated arguments equal the concatenation of all chunks for that id in order. The existing 4 static tests in `_accumulate_tool_calls_test.dart` cover corner cases but not random interleaving.
  **RESOLVED:** done — added a Glados interleaving property (numRuns 150) in the accumulate test file: seeds map to deltas across 3 interleaved tool indices with non-JSON fragments (so the replace-on-complete-JSON guard can't mask concatenation); asserts exactly one entry per unique effective id and per-id arguments equal to the in-order chunk concatenation.

- [ ] **[LOW]** `TaskSummaryRepository.parseLocalStartOfDay` / `parseLocalEndOfDay` are pure date-parsing functions. `task_summary_tool_model_test.dart` already has a Glados round-trip test for `TaskSummaryRequest` serialization (correct). A small Glados property could generate random date strings and assert that valid `YYYY-MM-DD` dates parse without throwing, while strings with wrong lengths or impossible days always throw `FormatException`.
