## Test execution speed opportunities

- [ ] **[HIGH]** `test/.../ui/pages/daily_os_page_test.dart` — `pumpAndSettle()` is called 20 times across 23 tests (every test uses it). For tests that only pump async providers (no animations), `await tester.pump()` (for microtasks) + `await tester.pump(Duration(milliseconds: 100))` (for provider resolution) would be faster and would eliminate the 10-second hang risk if a provider never settles. At minimum, the structure tests (lines 202–254) should use `tester.pump()` since they test static composition only.
- [ ] **[MED]** `test/.../ui/pages/set_time_blocks_page_test.dart` — `pumpPage` always calls `await tester.pumpAndSettle()` at line 153. The layout tests (lines 156–200) each call `pumpPage` and then do nothing that involves animations. Replace `pumpAndSettle()` in `pumpPage` with `tester.pump()` and a single extra `tester.pump()` for provider microtasks, or make `pumpAndSettle` opt-in.
- [ ] **[MED]** `test/.../ui/pages/set_time_blocks_page_test.dart:558` — Uses `DateTime.now()` in the "shows Today prefix when selected date is today" test to construct `todayDate`. Per AGENTS.md fake-time policy, tests must use deterministic dates. This test's correctness depends on `_date.dayAtMidnight == DateTime.now().dayAtMidnight` inside the impl, making it date-sensitive. The test can be made deterministic by injecting a fake "today" clock into the widget or by using a provider override for the current date comparison. Until then the test is potentially flaky if it runs near midnight.
