## Generative (Glados) testing opportunities

None of the files in this scope currently have Glados tests.

- [x] **[HIGH]** `lib/features/daily_os/ui/widgets/daily_timeline.dart` — `_slotContains`, `_groupNestedSlots`, `_assignLanes`, `_getLaneCount`, `_assignLanesToSlots` (lines 48–161, 953–1013) are pure functions operating on structured `ActualTimeSlot` inputs. Key invariants to probe:
  - `_assignLanes` never assigns two overlapping slots to the same lane (non-overlap invariant for all input sequences).
  - `_groupNestedSlots` never assigns the same slot as both a parent and a child.
  - `_getLaneCount` equals `max(laneIndex) + 1` for all non-empty inputs.
  - These are worth ~2–3 Glados property tests once moved to `lane_assignment_utils.dart`.

- [ ] **[HIGH]** `lib/features/daily_os/ui/widgets/time_budget_card.dart` — `_formatCompactDuration` (lines 21–29) is a pure `Duration → String` formatter. Invariants: for any `duration` with `inHours > 0` the output always starts with `${hours}h`; for `mins == 0` no "m" suffix appears; round-trip parsing is testable. A handful of Glados property tests over arbitrary `Duration` values would guard against future formatting regressions.
  - `_getTaskStatusColor` (lines 32–43): a pure `TaskStatus → Color` switch — all status enum values are covered, no nesting. Parameterized static tests are sufficient; not a Glados candidate.
  - `_calculatePriorityBadgeTop` (lines 971–976): pure arithmetic — `isCompleted × completedBadgeHeight + isDueOrOverdue × dueBadgeHeight + initialTop`. Algebraic invariant (additive, bounded). A single Glados property test over `(bool, bool)` input pairs.

- [ ] **[MED]** `lib/features/daily_os/ui/widgets/planned_block_drag_state.dart` — `PlannedBlockDragState.copyWith` (line 78): idempotence invariant — `state.copyWith() == state` for all valid states. `hasChanged` invariant — `hasChanged` iff `currentStart != originalStart || currentEnd != originalEnd`. ## Summary

- 2 oversized impl files (`daily_timeline.dart` 1203 ln, `time_budget_card.dart` 1015 ln); 2 severely oversized test files (`daily_timeline_test.dart` 2759 ln, `time_budget_card_test.dart` 2578 ln).
- 8 weak/smoke-only tests found (`findsOneWidget(WidgetType)` or `findsWidgets` as sole meaningful assertion): `time_budget_card_test` ×4, `daily_timeline_test` ×2, `daily_os_empty_states_test` ×1, `day_summary_test` ×1.
- 7 copy-paste test permutations in `time_budget_card_test.dart` (task-status check-icon tests) — convertible to a parameterized loop saving ~150 ln.
- 3 Glados candidates: lane-assignment pure functions (`daily_timeline.dart` — **HIGH**, ~2–3 properties), `_formatCompactDuration` and `_calculatePriorityBadgeTop` in `time_budget_card.dart` (**MED**), `PlannedBlockDragState.copyWith`/`hasChanged` (**MED**).
- 5 speed wins: `add_budget_sheet_test` (38 `pumpAndSettle`, highest risk), `daily_os_empty_states_test` (20), `draggable_planned_block_test` (16), `time_budget_list_test` (14), `category_block_row_test` (11).
- Biggest single opportunity: extract `_slotContains` / `_groupNestedSlots` / `_assignLanes` from `daily_timeline.dart` into a dedicated utility file — it simultaneously reduces the oversized impl, enables direct unit + Glados tests for non-trivial pure algorithms, and removes the dependency on full widget tests to cover algorithm correctness.
