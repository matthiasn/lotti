## Recurring patterns

- **Massively oversized impl files with clear split seams** — `day_agent_plan_service.dart` (1879 lines), `day_agent_workflow.dart` (1219 lines), `day_timeline.dart` (1832 lines) all exceed the 1000-line hard limit with multiple documented extraction seams. None have been split.
- **Copy-pasted setUp blocks** — 15 identical group-level setUp blocks in `real_day_agent_test.dart` and 8 inline `ProviderContainer` constructions in `day_agent_providers_test.dart` are the most egregious instances of the anti-pattern identified in AGENTS.md.
- **Inline DayAgentInterface re-implementations** — `_RecordingAgent` is independently reimplemented in full (~200 lines each) in at least 3 page test files (`capture_page_test`, `day_page_test`, `refine_page_test`). A shared factory would eliminate ~400 lines.
- **Pure functions with zero tests** — `categoryColorFromHex`, `day_agent_config.toJson`, `DayAgentKind.matches`, `_resolveRealtimeFinalTranscript` paths (1), (2), (3) are pure logic with no test coverage.
- **Fake-time policy violations** — `Future.delayed` drain loops, real `Timeout(seconds:)` annotations, and `tester.runAsync` real-I/O calls appear across `logic/`, `state/`, and `ui/pages/`.
- **Glados-eligible pure functions not yet property-tested** — `categoryColorFromHex`, `dailyOsNextRouteDate` round-trip, `_remainingScheduledWakeAt`, `positionForHourValue`/`fromBlocks` in `day_timeline` are all pure functions with clear algebraic invariants and no Glados coverage.
- **Missing test files for production widgets** — `category_chip.dart`, `live_waveform.dart`, `parsed_card.dart`, `why_chip.dart` in `ui/widgets/`; `day_agent_config.dart`, `day_agent_kind.dart` in `agents/domain/` — all have zero test coverage despite being active production code.
