  **RESOLVED — found a real bug:** the antisymmetry/oracle Glados property exposed a DST defect: `_daysBetween` used local `difference().inDays`, which under-counts by one across spring-forward (Jan 1 → Apr 1 measured 90 days, not 91). ## Test execution speed opportunities

- [x] **[HIGH]** `real_day_agent_test.dart` lines 209–294 (success path `draftDayPlan`) uses `await` on a real `Future.delayed` emitted by `MockDayAgent.parseCaptureToItems` inside the poll loop. The test carries `timeout: const Timeout(Duration(seconds: 10))`. The same issue applies to lines 1600–1699 (`proposePlanDiff` success path, `timeout: 5s`). Both tests must wait for the real 500 ms poll interval to elapse. Converting each to use `withClock(Clock.fixed(...))` with a `MockDayAgent` whose latencies are all `Duration.zero` plus `fakeAsync` to elapse the poll would reduce these tests from ≥500 ms to microseconds and make them deterministic. **Estimated saving: ~1 second per CI run.**

- [x] **[MED]** The 15 group-level setUp blocks each reconstruct mock objects and adapter on every test. While cheap per call, the volume (~50–60 test functions in 15 groups) means every test allocates 5+ mocks. Sharing a `_TestBench` (described above) would reduce allocations without any behavioural change.
  **RESOLVED (stale/assessed):** the shared `_TestBench.create()` scaffolding already exists and is used by every group; the per-test fresh allocations it produces are the deliberate isolation policy for the batched CI runner, and five mocktail mocks per test are negligible.

- [x] **[LOW]** `mock_day_agent_test.dart` `draftDayPlan` test (line 144) calls `await agent.draftDayPlan(...)` with `draftLatency: Duration.zero` so it is already fast. No action required.
  **RESOLVED — the observation was stale:** the `draftDayPlan` test (line 144) lives in the first `MockDayAgent` group whose `setUp` zeroed `parseLatency`/`pendingLatency`/`triageLatency` but **not** `draftLatency` — so the test was actually awaiting the default 400 ms real `Future.delayed`, violating the no-real-delay policy and adding ~400 ms of wall-clock per run. Added `draftLatency: Duration.zero` to that group's `setUp`, making the test deterministic and fast as the item assumed it already was.

- Biggest opportunity: extract a `_TestBench` from the 15 identical setUp blocks in real_day_agent_test.dart — this alone removes ~130 lines of duplication and makes the test file legible.
