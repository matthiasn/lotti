## Test quality improvements

- [ ] **[HIGH]** **Massively duplicated setUp blocks** in `real_day_agent_test.dart`. The same five-mock + adapter pattern appears in 15 of 15 `group` blocks (lines 34–46, 165–177, 369–381, 529–541, 651–663, 761–773, 905–917, 1102–1114, 1416–1428, 1489–1501, 1753–1765, 1906–1918, 2139–2151, 2202–2214, 2440–2452). Extract a shared `_TestBench` class with `captureService`, `planService`, `dayAgentService`, `journalDb`, and `adapter` fields, plus a `create({MockDayAgent? fallback})` factory. This eliminates ~130 lines of duplication and makes the file comply with the "mock setup must not dwarf test logic" rule.

- [ ] **[HIGH]** **Locally duplicated helper functions** — `buildDayPlan` appears in groups at lines 179 and 2216 with nearly identical bodies; `buildChangeSet` appears at lines 403 and 1527 with structurally identical patterns; `buildDiff` appears at lines 418 and 1767. These should be file-level private helpers, not per-group inner functions.

- [ ] **[MED]** `mock_day_agent_test.dart` line 213–224: the test `'DayAgentInterface is implementable; equality on value objects works'` mixes two unrelated concerns (interface implementability as a compile-time check and value-object equality). Split into two tests or drop the compile-time check (it adds no runtime value; the file already exercises the interface through `MockDayAgent` calls). The `_NullAgent` class (lines 561–693, ~133 lines) exists solely for one smoke test; consider replacing with `MockDayAgent` or removing if the interface check is dropped.

- [ ] **[MED]** **real_day_agent_test.dart timeout annotations** at lines 293 (`Timeout(Duration(seconds: 10))`), 1478, 1698, 1742 (`Timeout(Duration(seconds: 5))`): these tests call `await adapter.draftDayPlan(...)` / `await adapter.proposePlanDiff(...)` with real `Future.delayed` inside the poll loop. The two "success" path tests (lines 209–294 and 1600–1699) do await a real `Future.delayed(_draftPollInterval)` (500 ms) because the mock they use (`MockDayAgent`) calls `Future.delayed`. Those tests should use `fakeAsync` with `async.elapse(const Duration(milliseconds: 500))` instead of relying on real wall-clock time, removing the need for `Timeout` annotations.

- [ ] **[MED]** `day_agent_models_test.dart` covers only `CaptureId`, `DayAgentCategory`, `DraftPlan.copyWith`, and `TimeBlock.duration`. It does not exercise `PendingItem`, `TriageResult`, `ParsedItem`, `AgendaItem`, `PlanDiff`, `CompletedItem`, `CarryoverItem`, `ShutdownMetrics`, `TomorrowNote`, or any of the enum conversions. These are value objects whose correctness is assumed by the rest of the codebase.

- [ ] **[LOW]** `real_day_agent_test.dart` lines 2161–2192: the test `'mocked tools ... delegate to the fallback agent'` calls `surfaceShutdownData`, `recordReflection`, `recordCarryoverDecision`, `generateTomorrowNote`, and `surfaceTaskCorpus` in a single test and then only checks `isA<List<CompletedItem>>()` and `isNotEmpty` on the corpus. Each delegated method deserves its own focused test that verifies the delegation is exact (the real fallback is used, not some stub).

## Test execution speed opportunities

- [x] **[HIGH]** `real_day_agent_test.dart` lines 209–294 (success path `draftDayPlan`) uses `await` on a real `Future.delayed` emitted by `MockDayAgent.parseCaptureToItems` inside the poll loop. The test carries `timeout: const Timeout(Duration(seconds: 10))`. The same issue applies to lines 1600–1699 (`proposePlanDiff` success path, `timeout: 5s`). Both tests must wait for the real 500 ms poll interval to elapse. Converting each to use `withClock(Clock.fixed(...))` with a `MockDayAgent` whose latencies are all `Duration.zero` plus `fakeAsync` to elapse the poll would reduce these tests from ≥500 ms to microseconds and make them deterministic. **Estimated saving: ~1 second per CI run.**

- [ ] **[MED]** The 15 group-level setUp blocks each reconstruct mock objects and adapter on every test. While cheap per call, the volume (~50–60 test functions in 15 groups) means every test allocates 5+ mocks. Sharing a `_TestBench` (described above) would reduce allocations without any behavioural change.

- [ ] **[LOW]** `mock_day_agent_test.dart` `draftDayPlan` test (line 144) calls `await agent.draftDayPlan(...)` with `draftLatency: Duration.zero` so it is already fast. No action required.

## Summary

- 2 impl files oversized (real_day_agent.dart 848 lines, mock_day_agent.dart 825 lines); 1 borderline (day_agent_models.dart 644 lines).
- 1 test file massively oversized (real_day_agent_test.dart 2,558 lines — largest in scope by far).
- 15 copy-pasted setUp blocks in real_day_agent_test.dart; 3+ duplicated local helper functions — all reducible to a single `_TestBench` + file-level helpers.

- 2 speed wins: real-delay tests at lines 209 and 1600 add ≥1 second of real wall-clock time per CI run — both are avoidable with `fakeAsync` + zero-latency mock.
- Biggest opportunity: extract a `_TestBench` from the 15 identical setUp blocks in real_day_agent_test.dart — this alone removes ~130 lines of duplication and makes the test file legible.
