## Generative (Glados) testing opportunities

- [ ] **[HIGH]** `categoryColorFromHex` in `category_color.dart` (line 10) is a pure function that maps `String → Color`. Two Glados properties are natural fits:
  1. *Round-trip identity* — for any valid 6-hex-char string (lowercase or uppercase), `categoryColorFromHex(hex).value & 0x00FFFFFF == int.parse(hex, radix: 16)`.
  2. *Length-trim invariant* — for any string `s` whose first 6 chars form a valid hex value, `categoryColorFromHex(s)` equals `categoryColorFromHex(s.substring(0, 6))`.

  Both properties require only a simple `any.string` generator with a small `ExploreConfig(numRuns: 100)`. Tag with `tags: 'glados'` as required.

- [ ] **[HIGH]** `dailyOsNextRouteDate` / `parseDailyOsNextRouteDate` in `daily_os_next_routes.dart` have a clean round-trip property: for any `DateTime d`, `parseDailyOsNextRouteDate(dailyOsNextRouteDate(d))` must equal `DateTime(d.year, d.month, d.day)` (local midnight, not the original time-of-day). The inverse is also interesting: any string that `parseDailyOsNextRouteDate` returns non-null for must round-trip back to the same string via `dailyOsNextRouteDate`. These properties robustly exercise year/month/day boundary arithmetic without hand-picking examples. Use `any.dateTime` from the existing shared generators.

- [ ] **[MED]** `dailyOsNextRoutePath` has a structural invariant: for any `(target, date)`, the returned path always matches the pattern `/calendar/<target.name>/<YYYY-MM-DD>`. A Glados generator over `DailyOsNextRouteTarget.values` and `any.dateTime` would verify this without requiring exhaustive hand-rolled cases.

## Coverage / missing-behavior gaps

- [ ] **[HIGH]** `category_color.dart` — `categoryColorFromHex` has zero test coverage:
  - Happy path: 6-char hex without `#` prefix (e.g., `'5ED4B7'`).
  - `#`-prefixed input: `'#5ED4B7'` should strip the prefix and parse correctly.
  - Overlong input (>6 hex chars, e.g., `'5ED4B7FF'`): only first 6 chars should be used.
  - Malformed input (e.g., `'ZZZZZZ'`, `''`): must return `Colors.grey` without throwing.
  - Alpha channel forced to 0xFF: for any valid input, the result should always have `alpha == 0xFF`.

- [ ] **[MED]** `daily_os_next_routes_test.dart` — the `parseDailyOsNextRouteDate` test does not cover:
  - Day `00` (invalid zero day) — the `DateTime` overflow/rejection path.
  - Leap-year boundary: `'2024-02-29'` should parse successfully; `'2025-02-29'` should return null.
  - Negative numbers sneaking through a non-`\d` match are already blocked by the `^(\d{4})-(\d{2})-(\d{2})$` regex, but `'2026-00-01'` and `'2026-01-00'` are valid regex matches that should be rejected by the overflow check — neither is currently tested.

- [ ] **[LOW]** `dailyOsNextRoutePath` — none of the three enum values (`refine`, `commit`, `shutdown`) have separate tests verifying that the `target.name` segment is exactly the lowercase enum member name. The current test exercises all three but bundles them in one `test(...)` call; a failure message would not name which target broke.
