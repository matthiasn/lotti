## Coverage / missing-behavior gaps

- [x] **[HIGH]** `category_color.dart` — `categoryColorFromHex` has zero test coverage:
  - Happy path: 6-char hex without `#` prefix (e.g., `'5ED4B7'`).
  - `#`-prefixed input: `'#5ED4B7'` should strip the prefix and parse correctly.
  - Overlong input (>6 hex chars, e.g., `'5ED4B7FF'`): only first 6 chars should be used.
  - Malformed input (e.g., `'ZZZZZZ'`, `''`): must return `Colors.grey` without throwing.
  - Alpha channel forced to 0xFF: for any valid input, the result should always have `alpha == 0xFF`.

- [x] **[MED]** `daily_os_next_routes_test.dart` — the `parseDailyOsNextRouteDate` test does not cover:
  - Day `00` (invalid zero day) — the `DateTime` overflow/rejection path.
  - Leap-year boundary: `'2024-02-29'` should parse successfully; `'2025-02-29'` should return null.
  - Negative numbers sneaking through a non-`\d` match are already blocked by the `^(\d{4})-(\d{2})-(\d{2})$` regex, but `'2026-00-01'` and `'2026-01-00'` are valid regex matches that should be rejected by the overflow check — neither is currently tested.
  **RESOLVED:** Added two tests: 'accepts a leap-year Feb 29 but rejects it in a common year' (`'2024-02-29'` → `DateTime(2024, 2, 29)`; `'2025-02-29'` → null) and 'rejects regex-valid but calendar-invalid zero components' (`'2026-01-00'` → null via month/year guard; `'2026-00-01'` → null via year guard). Behaviors verified against the source's `parsed.year/month/day` equality guard.

- [x] **[LOW]** `dailyOsNextRoutePath` — none of the three enum values (`refine`, `commit`, `shutdown`) have separate tests verifying that the `target.name` segment is exactly the lowercase enum member name. The current test exercises all three but bundles them in one `test(...)` call; a failure message would not name which target broke.
  **RESOLVED:** Split the bundled test into a `for (final target in DailyOsNextRouteTarget.values)` loop generating one named test per target ('builds a $target route path with the lowercase enum name'). Each asserts the full path, that `target.name == target.name.toLowerCase()`, and the exact segment split — so a failure names the broken target. Avoids copy-paste permutations per the test-quality rules.
