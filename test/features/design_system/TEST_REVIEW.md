## Recurring patterns

- **Untested production files in `components/`** — 5 implementation files across `lists/` and `navigation/` subdirs have zero test coverage: `grouped_card_row_surface.dart`, `design_system_grouped_list.dart`, `design_system_list_palette.dart`, `design_system_showcase_mobile_chrome.dart`, `desktop_detail_empty_state.dart`.
- **pumpAndSettle overuse in task_filter tests** — the task_filter subdir alone has ~24 replaceable `pumpAndSettle` calls across 4 test files; the pattern of calling `pumpAndSettle` unconditionally in pump helpers (even for tests that only check static visibility) is the root cause.
- **Dark-theme coverage gap** — most component test files only test `DesignSystemTheme.light()`; only `typography_widgetbook_test.dart` and `task_filter_widgetbook_test.dart` include dark-theme tests. Buttons, checkboxes, breadcrumbs, captions, context menus, calendar pickers, file uploads, and headers all lack dark-theme token assertions.
- **Glados-eligible pure functions without property tests** — `DesignSystemTaskFilterState` toggle/clear operations, `stripTrailingColon`, `monoMetaStyle`, `DsTokens.lerp`, `_parseWidth` are all non-trivial pure functions with clear algebraic invariants and no Glados coverage.
- **Smoke-only assertions in widgetbook tests** — ~10 thin widgetbook test files assert only component name, use-case name, and widget-count presence. Interactive states (hover, disabled, tap) are not exercised. The branding widgetbook test is the most egregious: sole assertion is `findsOneWidget`.
- **Copy-paste test permutations** — `design_system_task_list_item_test.dart` has 4 near-identical per-priority test bodies; `design_system_toggle_test.dart` has 2 near-identical AnimatedContainer dimension tests; `design_system_radio_button_test.dart` has repeated token-extraction patterns — all should be parameterized loops.
- **Missing test files for theme utilities** — `typography_helpers.dart` and `breakpoints.dart` in `theme/` have no test files despite containing behavioral logic (`monoMetaStyle`, `isDesktopLayout`) that is exercised across the app.
