## Summary

- 1 oversized test file (`habits_controller_test.dart`, 779 lines) and 1 near-oversized test file (`habit_completion_card_test.dart`, 581 lines) warrant splitting along their existing test group boundaries.
- 3 impl files have **no test file**: `habit_page_app_bar.dart` (conditional icon rendering), `status_segmented_control.dart` (4-segment filter), and `habit_streaks.dart` (completion count display).
- Inline `getIt` boilerplate appears in 3 test files (`habit_settings_controller_test.dart`, `habits_filter_test.dart`, `habits_search_test.dart`) — all should migrate to `setUpTestGetIt()`/`tearDownTestGetIt()`.
- 4 strong Glados candidates exist in pure functions in `habits_state.dart` (`completionRate`, `habitMinY`, `activeBy`) and the streak-counting logic in `habits_controller.dart`; zero Glados tests currently exist for this feature.
- `pumpAndSettle` is overused in search (×9), page (×10), filter (×7), and card (×5) tests — all on non-animated or bounded-animation widgets; replacing with `tester.pump()` or bounded pumps is the highest-ROI speed improvement.
- The biggest single opportunity is extracting the `MockHabitsController` defined inline in `habits_tab_page_test.dart` into `test/mocks/mocks.dart`, which would allow the inline controller stubs in filter/search tests to be consolidated too.
