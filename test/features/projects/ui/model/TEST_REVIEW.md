## Generative (Glados) testing opportunities

- [x] **[HIGH]** `ProjectsQuery.matchesCategory` + `ProjectsQuery` equality/hashCode — This is a pure value class with a `Set<String>` field and a simple predicate. Glados properties: **RESOLVED:** the empty-set/contains properties already existed in `projects_overview_models_test.dart`; this pass adds the missing set-equality property — `Glados<Set<String>>` (120 runs) asserting reversed-insertion-order sets stay equal with matching hashCodes and a widened set breaks equality.
  - Empty `categoryIds` always returns true for any input (including null).
  - Non-empty `categoryIds` returns true iff the input is in the set.
  - `equals(a, b)` iff `a.categoryIds == b.categoryIds` (set equality, not order-sensitive).
  A `Glados2(any.string, any.listOf(any.string))` test with 80 runs would replace 8 hand-crafted example tests.

- [x] **[HIGH]** `applyProjectsFilter` round-trip / idempotence — Properties: **RESOLVED:** the identity-filter and bounded-count properties already existed; this pass adds the idempotence property (`apply(apply(s,f),f)` shape-equals `apply(s,f)` over generated filters, 120 runs) and the all-status-ids ≡ identity example test.
  - `applyProjectsFilter(snapshot, const ProjectsFilter())` returns groups with the same total count as the input (identity filter).
  - `applyProjectsFilter(applyProjectsFilter(snapshot, f), f)` returns the same result as `applyProjectsFilter(snapshot, f)` (idempotence).
  - With a filter that matches all status IDs, result equals the identity-filter result.
  These hold for any `ProjectsFilter` shape; a `Glados` generator for `ProjectsFilter` (combining `any.bool` flags for each status and arbitrary category ID strings) would cover the combinatorial space the manual tests cannot.

- [ ] **[MED]** `ProjectTaskRollupData.completionRatio` — The formula is `completedTaskCount / totalTaskCount`. Glados property: `completionRatio` is always in [0.0, 1.0] for non-negative inputs; `totalTaskCount == 0` always yields 0.0. A `Glados2(any.nonNegativeInt, any.nonNegativeInt)` test with 60 runs would cover edge cases (very large counts, equal values, zero).

- [ ] **[LOW]** `ProjectsFilter` equality/hashCode — Glados can verify reflexivity, symmetry, and transitivity of the custom `==` and `hashCode` (which uses `{...}.hashCode` over a `Set`). 40 runs; replaces the 7 manual equality tests.
