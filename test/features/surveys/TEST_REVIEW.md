## Test quality improvements

- [ ] **[LOW]** `test/features/surveys/ui/fill_survey_page_test.dart:69` — `expect(rpuiTask.onCancel, isNotNull)` is a bare existence check. Replace with a meaningful assertion: call `rpuiTask.onCancel!(null)` and verify the log emits `'No result'` (the `null` branch is already tested later; the inline `isNotNull` adds nothing).
- [ ] **[LOW]** `test/features/surveys/tools/run_surveys_test.dart:45` — `pumpRunner` calls `tester.pumpAndSettle()` immediately after `tester.tap()`. Because the modal is shown via a `Future<void>` and the research_package widget tree may have open animations, this is probably correct here. However, if the modal uses an animation that never settles, this will hang for the default 10 s. Prefer `tester.pump(const Duration(milliseconds: 300))` or use `pumpAndSettle` with a short explicit duration cap to make the intent explicit.

It does **not** verify that the `resultCallback` closure passed to `SurveyWidget.task` will call `persistenceLogic.createSurveyEntry` when invoked. A stub test driving `surveyWidget.onSubmit!(someResult)` and verifying the mock capture would close this behavioral gap without adding a new file.
