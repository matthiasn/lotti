## Test quality improvements

- [x] **[HIGH]** `backfill_request_service_test.dart` line 50–52 defines `_MockQueuePipelineCoordinator extends Mock implements QueuePipelineCoordinator` inline. `MockQueuePipelineCoordinator` already exists in `test/mocks/mocks.dart` (line 702). The inline version (`_MockQueuePipelineCoordinator`) is structurally identical and should be replaced with the centralized import. The same pattern — and the `MockInboundQueue` usage — is already wired correctly in `mocks.dart`. This is a direct violation of the "Use centralized mocks" rule.

- [x] **[HIGH]** `backfill_response_handler_test.dart` lines 1150, 1204, and 1225 directly use `DateTime.now()` in test assertions and state setup: **RESOLVED:** the three sites now use fixed dates under `withClock(Clock.fixed(...))` (the handler reads `clock.now()` since the companion lib fix), so the cooldown/rate-window assertions are deterministic.
  - Line 1150: `handler.recentlyResponded['$aliceHostId:3'] = DateTime.now();`
  - Line 1204: `..windowStart = DateTime.now()`
  - Line 1225: `final now = DateTime.now();`
  These three tests rely on real wall-clock time rather than `withClock(Clock.fixed(...), ...)` or `fakeAsync` with fake time. The fake-time policy says "Never use `DateTime.now()` in tests." The cooldown and rate-limit tests are exercising time-sensitive logic; they should use a fixed clock so they are deterministic and do not flake on slow CI runners. The `BackfillResponseHandler` itself calls `DateTime.now()` in `_isRecentlyResponded`, `_recordResponse`, `_cleanExpiredCooldowns`, and `_isRateLimited` — these could be injected via `package:clock` (already a dependency in the impl layer) so tests can pin them.

- [ ] **[MED]** Several tests in `backfill_request_service_test.dart` construct `BackfillRequestService` with nearly identical boilerplate (sequence service, sync database, outbox, vc service, logging, plus optional params). The per-test construction is 7–10 lines in 30+ tests. Extracting a `_buildService({...optional overrides})` factory would cut ~200 lines of repetition and make test intent clearer.

- [ ] **[MED]** `backfill_response_handler_test.dart` groups `handleBackfillRequest - EntryLink` (line 1491), `handleBackfillRequest - AgentEntity` (line 1824), `handleBackfillRequest - AgentLink` (line 2017) each repeat the same `when(mockSequenceService.getEntryByHostAndCounter...).thenAnswer` + `when(outboxService.enqueueMessage...).thenAnswer` block.
