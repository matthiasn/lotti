## Test execution speed opportunities

- [x] **[HIGH] `smart_journal_entity_loader_test.dart` — real filesystem I/O in every test** — All 29 **RESOLVED:** one `createTemp` in `setUpAll` (deleted in `tearDownAll`); each test now gets a counter-named subdirectory via `createSync()`. The setUp also moved from raw `getIt.reset()`/`allowReassignment` to `setUpTestGetIt(additionalSetup: ...)`.
  tests use `Directory.systemTemp.createTemp(...)` and write real files. This is necessary for the
  loader (it relies on real `File` operations), but the per-test `setUp`/`tearDown` creating and
  deleting temp directories adds real wall-clock time. Use a shared `setUpAll`/`tearDownAll` where
  the temp directory is clean-started once and individual tests write to distinct subdirectories via
  unique sub-paths, avoiding 29 OS directory operations. Estimated: ~200–400 ms reduction per run.

  `Future.delayed(Duration.zero)` in tests without `fakeAsync`** — Real event-loop yields introduce
  OS scheduling jitter. Per `test/README.md`, replace with `async.flushMicrotasks()` inside
  `fakeAsync` and `sync_engine_test.dart` line 217 similarly. Combined: low individual impact but
  contributes to suite non-determinism.

- [ ] **[MED] `read_marker_service_test.dart` Glados `numRuns: 180`** — This is at the high end of the
  180-run budget. The scenario space (7 orthogonal dimensions) is well-covered; `numRuns: 120`

- [ ] **[LOW] `session_manager_test.dart` Glados `numRuns: 180`** — Same rationale as above; 6

## Summary

- **1 oversized test file**: `smart_journal_entity_loader_test.dart` (1848 lines) — copy-paste
  boilerplate across 29 tests is the primary driver; a `_wireRoomChain` helper would cut ~200 lines.
- **1 structural violation**: `room_test.dart` is a second test file for `sync_room_manager.dart`;
  it must be merged and deleted (one-test-file-per-source-file rule).
- **6+ files with inline mock classes** that duplicate or shadow centralized mocks in
  `test/mocks/mocks.dart` (`MockClient`, `MockMatrixSyncGateway`, `MockSyncRoomManager`, etc.).
- **1 completely untested impl file**: `last_read.dart` (22 lines, 4 functions including
  `isServerAssignedMatrixEventId` and a silent `int.tryParse` path).
- **2 Glados tests missing `tags: 'glados'`** in `timeline_ordering_test.dart` — routing CPU-bound
  sort properties through the fast-feedback CI shard.
- **4 real `Future.delayed(Duration.zero)` waits** across `room_test.dart` (3) and
  `sync_room_manager_test.dart` (1) that should be `fakeAsync`/`flushMicrotasks`.
- **Biggest single opportunity**: Merging `room_test.dart` into `sync_room_manager_test.dart` and
  extracting the `_wireRoomChain` / entity-factory helpers in `smart_journal_entity_loader_test.dart`
  would together eliminate ~600 lines of redundancy, fix the structural violation, and resolve
  the most impactful quality issue in this scope.
