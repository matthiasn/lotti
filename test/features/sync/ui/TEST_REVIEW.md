## Summary

- `backfill_settings_page.dart` is the single biggest opportunity: oversized impl (1170 L) +
  oversized test (1352 L) + 78 `pumpAndSettle` + wrong test helper.
- 1 impl file oversized (backfill_settings_page.dart ≥ 1000 L); 1 test file oversized (backfill_settings_page_test.dart ≥ 1000 L).
- ~7 weak-assertion instances across 5 files (findsOneWidget-only or findsNothing-only assertions without interaction verification).

- ~120+ `pumpAndSettle` calls across root test files (78 in backfill alone) that could be converted to `tester.pump(duration)` for safer, faster tests.
