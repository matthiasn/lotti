## Summary

- `backfill_settings_page.dart` is the single biggest opportunity: oversized impl (1170 L) +
  oversized test (1352 L) + 78 `pumpAndSettle` + wrong test helper.
- 1 impl file oversized (backfill_settings_page.dart ≥ 1000 L); 1 test file oversized (backfill_settings_page_test.dart ≥ 1000 L).
- ~6 weak-assertion instances across 4 files (findsOneWidget-only or findsNothing-only assertions without interaction verification) — `sync_settings_page_test.dart` was deleted when mobile Sync moved to the shared settings tree.
- 2 Glados candidates: `_formatCount` (pure locale formatter) and `_buildActions` count invariants.
- ~110+ `pumpAndSettle` calls across root test files (78 in backfill alone) that could be converted to `tester.pump(duration)` for safer, faster tests (the ~9 in the deleted `sync_settings_page_test.dart` no longer count).
- `sync_modal_progress_test.dart` has been merged into `sync_modal_test.dart` and removed (one-test-file-per-source-file rule now satisfied).
