## Coverage / missing-behavior gaps

- [ ] **[HIGH]** `lib/features/whats_new/state/whats_new_controller.dart:119-137` — `_isNewerVersion` is the version-gating logic that prevents showing releases newer than the installed app. It is only tested indirectly: the mock version is set to `99.99.99` so the filter never fires. No test verifies that a release with `version = "100.0.0"` is skipped when installed version is `"0.9.980"`. A dedicated test (or the Glados property above) is needed.
- [ ] **[HIGH]** `lib/features/whats_new/repository/whats_new_service.dart:63-91` — three distinct error paths (`TimeoutException`, `SocketException`, `FormatException`) in `fetchIndex` and `fetchContent` are all caught and logged. The service test file (366 lines) should verify that each exception type results in `null` being returned and the `DomainLogger.error` call being made. Check whether `whats_new_service_test.dart` covers these paths.
- [ ] **[MED]** `lib/features/whats_new/ui/whats_new_modal.dart:123-128` — `markViewedAsSeen()` tracks `maxViewedIndex` and marks only pages 0..maxViewedIndex as seen on normal close. No test verifies the boundary where the user views only the first of three releases (maxViewedIndex stays 0) and closes — only the first release should be marked seen. The test at line 306 ("closing modal normally marks only viewed releases as seen via markAsSeen") covers the single-release case; add a multi-release case where the user does NOT navigate.
- [ ] **[MED]** `lib/features/whats_new/ui/whats_new_modal.dart:63-66` — `_extractImageUrls` is a static helper on `WhatsNewModal`. Its regex (`_imageUrlPattern`) is tested indirectly via the precaching test, but the function itself is not unit-tested for edge cases (empty markdown, markdown with no images, markdown with data-URI images that should not be precached via `NetworkImage`).

No test verifies that unrelated `SharedPreferences` keys are preserved. The existing `resetSeenStatus` test at line 280 does check `other_key` — this gap is actually covered. No action needed.

## Test execution speed opportunities

- [ ] **[HIGH]** `test/features/whats_new/ui/whats_new_modal_test.dart` — 29 `pumpAndSettle()` calls across 15 `testWidgets`. `WoltModalSheet` uses animated transitions so settle is required on open/close. However the `AnimatedContainer` indicator-dot test (line 125-144) builds the full modal and calls `pumpAndSettle` twice (open + navigate). The `AnimatedContainer` has a 300ms curve (`Curves.easeOutQuart`); a `tester.pump(const Duration(milliseconds: 350))` would be more deterministic than `pumpAndSettle` (which stops when animations quiesce but silently extends if they don't). Consider replacing `pumpAndSettle` with explicit pump durations in the navigation tests. **Estimated: −2s across the 15 tests.**
- [ ] **[MED]** `test/features/whats_new/state/whats_new_controller_test.dart` — no `pumpAndSettle` (pure provider tests). Speed is already good. The 6 inline container re-creations each trigger a fresh `SharedPreferences.setMockInitialValues` + provider build; consolidating into a helper will not measurably improve speed but reduces boilerplate maintenance cost.
- [ ] **[LOW]** `test/features/whats_new/util/whats_new_markdown_parser_test.dart` — Glados `numRuns: 160` on the `whatsNewMarkdown` generator which combines a list of up to 5 section images. The generator's `listWithLengthInRange(0, 5, imagePath)` means up to 5 `combine2` invocations per run. CPU cost is acceptable; no change needed.

## Summary

- 1 oversized impl file: `whats_new_modal.dart` (747 lines, exceeds 500-line soft ceiling); natural split into banner, navigation-footer, and coordinator files.
- 0 oversized test files by line count, but `whats_new_controller_test.dart` has 6 copy-paste container re-creations.
- 2 violations of centralized-mocks rule: inline `MockWhatsNewService` and `FakeWhatsNewRelease` in `whats_new_controller_test.dart`.
- 1 strong Glados candidate: `_isNewerVersion` version comparison logic — currently tested only via a `99.99.99` fixture that never exercises the filter.
- 2 meaningful coverage gaps: `_isNewerVersion` filter not tested with releases that are genuinely newer than installed version; `markViewedAsSeen` multi-release partial-view scenario not tested.
- Biggest opportunity: split `whats_new_modal.dart` into 3 files to get under the 500-line ceiling, and add a direct Glados test for `_isNewerVersion` — the only non-trivial pure logic in the feature that currently has no generative coverage.
